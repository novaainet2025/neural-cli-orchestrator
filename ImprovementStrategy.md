## 핵심 개선 목표 (3가지)

1. **신뢰성 강화** – 거짓 보고와 실패 작업을 최소화하여 실제 성공률을 95% 이상으로 끌어올린다.
2. **자동화 확대** – 반복적인 검증·보고·문서 동기화를 자동화해 인적 개입을 50% 이상 감소시킨다.
3. **측정 체계 고도화** – KPI 정의와 대시보드 구축으로 정량적인 성과를 가시화한다.

## 구체적 실행 계획 (각 목표별)

### 목표 1: 신뢰성 강화
- **원인 분석**: `tasks_failed` 로그와 `false_report_count` 메타 데이터를 정기적으로 파싱·분류.
- **리트라이·폴백**: 실패 작업에 대해 exponential backoff 재시도 로직 적용 (3회 재시도 후 대체 에이전트 전환).
- **거짓 보고 차단**: 작업 완료 전 T1 레벨 검증(gemma‑gate‑check) 통과 시에만 `tasks_completed` 카운트.
- **성과 지표**: 실패 비율 <5%, 거짓 보고 <2건 목표.

### 목표 2: 자동화 확대
- **자동 테스트 파이프라인**: 작업 완료 직후 `npm run test:run` 실행 및 결과를 DB에 기록.
- **Obsidian 문서 자동 동기화**: 새로운 개선 노트가 생성될 때 Git 커밋·PR 자동 생성 (`gh pr create`).
- **실시간 상태 보고**: 매 1시간마다 현재 `tasks_*` 메트릭을 Slack/Webhook 으로 전송.
- **구현 위치**: `src/core/TaskScheduler.ts`에 스케줄러, `src/utils/ObsidianSync.ts` 및 `src/integrations/Notifier.ts` 모듈 추가.

### 목표 3: 측정 체계 고도화
- **핵심 KPI**: 성공률, 평균 작업 처리 시간, 거짓 보고 건수, 자동화 커버리지.
- **대시보드**: Fastify 엔드포인트 `/metrics` 에 JSON 제공, Grafana 연동.
- **Obsidian 메트릭 페이지**: `Metrics.md` 자동 업데이트 스크립트.
- **검증 루프**: 사이클 종료 시 `npm run test:run && curl http://localhost:6200/health` 로 전반적인 상태 검증.

## 자동화 가능한 부분
- 작업 리트라이 로직 (중앙 `RetryHandler` 모듈)
- 테스트 실행 및 결과 수집 CI 스크립트
- 문서 커밋/PR 자동화 (GitHub CLI)
- 실시간 메트릭 전송 (Prometheus exporter)
- 거짓 보고 검증 (T1 파일/DB 상태 확인 후 보고)
