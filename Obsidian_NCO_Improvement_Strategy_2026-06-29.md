- 완료: 84건 (84%)
- 실패: 12건 (12%)
- 정체: 4건 (4%)

- Obsidian 문서: 10,334개, 개선 노트: 991개 (작업 대비 10~100배 적체)

## 핵심 개선 목표 (3가지)
1. **허위 완료 보고 감소** – 보고된 성공을 실제 검증된 성공으로 정밀화
2. **실패·정체 최소화** – 작업 안정성 및 재시도 메커니즘 강화
3. **지식·개선 노트 적체 해소** – Obsidian과 개선 노트 흐름을 자동화·우선순위화

## 구체적 실행 계획
### 목표 1: 허위 완료 보고 감소
- **품질 게이트 Hook**(`nco-stop-quality-gate.sh`) 도입
  - 작업 완료 전 `tsc --noEmit` 및 전체 테스트(`npm run test:run`) 실행
  - 테스트/컴파일 실패 시 작업 상태를 `failed` 로 전환하고 자동 재오픈
- **신뢰 점수 시스템** 구축
  - 에이전트별 허위 보고 비율을 추적·점수화
  - 낮은 점수 에이전트는 우선순위 낮춤·재교육 트리거
- **리포트 검증 파이프라인**
  - `ollama` 검증 단계에서 결과 요약과 실제 파일/DB 상태 T1 검증 추가

### 목표 2: 실패·정체 최소화
- **하트비트·타임아웃 정책**
  - `stuck` 작업에 60s 하트비트 없음 시 자동 `failed` 전환
  - 재시도 큐에 재삽입, 최대 3회 재시도 후 알림
- **OrchestratedLoop 개선**
  - 무한 대기 루프 방지용 `AbortController`와 타임아웃 적용
  - 로그 레벨 확대, `error` 로그에 작업 ID 포함
- **실패 원인 자동 분류**
  - `cursor-agent` 리뷰에서 `failure reason` 태그 추출
  - 주요 패턴(CLI hang, 타임아웃, 테스트 실패)별 자동 개선 스크립트 매핑

### 목표 3: 지식·개선 노트 적체 해소
- **주간 배치 파이프라인**
  - `Obsidian` 폴더를 스캔 → `improvement_notes`와 매칭
  - 중복·낮은 우선순위 노트 자동 폐기·아카이브
  - 남은 노트를 NCO Commander 작업(`nco_task opencode`) 형태로 변환
- **자동 태스크 생성**
  - `nco_parallel`을 이용해 `codex`와 `cursor-agent`에게 동시에 구현·리뷰 할당
  - 태스크 메타데이터에 `obsidian_id` 포함 → 완료 시 Obsidian에 체크 표시 자동화
- **Obsidian ↔ NCO 연동 스크립트**
  - `obsidian-sync.sh` (Node) : 마크다운 메타데이터 ↔ DB 동기화
  - 성공 시 Git 커밋 및 PR 자동 생성 (`gh pr create`)

## 자동화 가능한 부분
- 품질 게이트 Hook 실행 및 결과 집계 → CI 파이프라인 자동화
- 하트비트·타임아웃 감시 → `nco-monitor` 서비스 스크립트
- 실패 원인 라벨링 및 분류 → `cursor-agent` 리뷰 결과 파싱 스크립트
- Obsidian 노트 스캔·우선순위 정렬 → Node/TS 스크립트 (`obsidian-sync.ts`)
- 태스크 → PR 자동 생성 및 마크다운 체크박스 업데이트 (`gh` CLI 사용)
