## 핵심 개선 목표 (3가지)
1. **작업 정착성 향상** – 멈춘 작업을 최소화하고 100% 성공률 달성
2. **거짓 보고 감소** – 시스템 신뢰성을 높이기 위해 거짓 보고를 0에 가깝게 유지
3. **지식 관리 효율화** – Obsidian 문서와 개선 노트의 활용 및 자동화를 강화

## 구체적 실행 계획
### 목표 1: 작업 정착성 향상
- 스테이지별 타임아웃 및 재시도 로직 강화
- Stuck 작업 감지 알림 추가 (Redis pub/sub → WebSocket UI)
- 큐 모니터링 대시보드에 Stuck 비율 시각화
### 목표 2: 거짓 보고 감소
- False report 검증 파이프라인 도입 (다중 검증 로그 + 샘플링)
- 거짓 보고 발생 시 자동 티켓 생성 및 담당자 알림
- 정기적인 거짓 보고 리뷰 회의 스케줄링 (주간)
### 목표 3: 지식 관리 효율화
- Obsidian 문서 자동 태깅 스크립트 구현 (`obsidian-watcher.sh` 확장)
- 개선 노트 자동 요약 후 주요 이슈 대시보드에 반영
- 새 문서/노트 생성 시 메타데이터(작성자, 일시) 자동 삽입

## 자동화 가능한 부분
- **Stuck 작업 자동 복구**: 일정 시간 이상 멈춘 작업을 자동 재시도 스크립트
- **거짓 보고 자동 감지**: 로그 패턴 기반 AI 검증 모델 호출
- **Obsidian 메타데이터 자동 삽입**: `obsidian-context-builder.sh`에 후크 추가
- **주간 개선 노트 집계**: CI 파이프라인에서 `npm run summarize-notes` 실행
