## 구체적 실행 계획
### 목표 1: 작업 완료율 향상
- **원인 분석**: 작업 중단, 리소스 부족, 의존성 지연 파악 (주간 로그 검토).
- **자동 재시도 로직**: `src/core/task-manager.ts`에 실패/멈춤 작업 자동 재시도 스케줄러 추가 (exponential backoff).
- **우선순위 큐**: 높은 중요도 작업을 먼저 처리하도록 작업 큐에 우선순위 레벨 적용.
- **성능 모니터링**: Redis‑SQLite 동기화 주기를 30s → 10s 로 축소, 병목 감시.

### 목표 2: 스택 작업 해소
- **스택 감지**: `src/core/observer.ts`에 스택 감시 타이머(5분 이상 미진행 시 알림) 구현.
- **자동 복구**: 스택 작업 발견 시 동일 컨텍스트 복제 후 재시도, 최대 3회 시도 후 관리자 알림.
- **인증/샌드박스 강화**: `src/security/`에서 오래된 락 해제 로직 추가.

### 목표 3: 거짓 보고 감소
- **검증 강화**: 모든 보고 단계에 T1 검증(파일/DB 상태 확인) 삽입.
- **거짓 보고 로그**: `src/utils/false-report-logger.ts` 도입, 발생 시 `false_report_count` 자동 감소.
- **리뷰 프로세스**: `cursor-agent` 자동 코드 리뷰 후 `ollama` 검증 단계 삽입.

## 자동화 가능한 부분
- **주간 성능 보고**: `cron` 잡 (`npm run report:weekly`) → markdown 자동 생성 및 Obsidian vault에 커밋.

- **거짓 보고 자동 감축**: `false-report-logger`가 감지 시 자동 `git commit` 및 PR 생성.
