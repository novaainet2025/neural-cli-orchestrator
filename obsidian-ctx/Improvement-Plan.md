## 핵심 개선 목표 (3가지)

1. **허위 보고 감소** – 실제 완료와 보고된 완료 간 격차 해소
2. **정체 작업 해소** – 스택에 남은 작업의 진행 속도 향상
3. **자동화 확대** – 문서화·노트 생성·지표 수집 자동화

## 구체적 실행 계획 (각 목표별)

### 1. 허위 보고 감소
- **검증 게이트** 도입: 작업 완료 전 자동 테스트·산출물 존재·API 응답 확인
- **오류 재할당** 로직: 검증 실패 시 `failed` 로 전환 후 재시도 큐에 삽입
- **모니터링 대시보드**에 허위 보고 비율 시각화

### 2. 정체 작업 해소
- **우선순위 재조정**: `tasks_stuck` 태스크에 높은 우선순위 플래그 부여
- **에이전트 풀 확대**: 현재 `cursor-agent`만 작동 중임을 고려해 `opencode`·`mlx` 복구·추가 에이전트 활성화
- **타임아웃 알림**: 정해진 시간(예: 30분) 내 진행되지 않으면 관리자 알림

### 3. 자동화 확대
- **Obsidian 자동 노트 생성** 스크립트(`obsidian-context-builder.sh`)에 주기적 실행 추가
- **지표 수집 파이프라인**(`obsidian-watcher.ts`)에 `false_report_count`, `tasks_stuck` 등 신규 메트릭 포함
- **CI/CD 파이프라인**에서 `npm run test:run` 후 자동 리포트 생성 및 Obsidian에 커밋

## 자동화 가능한 부분
- 테스트·검증 단계 자동화 (GitHub Actions)
- 정체 작업 알림 Slack/Discord webhook 연동
- Obsidian 노트 템플릿 자동 업데이트 (Markdown 템플릿 엔진 사용)
- 성능 메트릭 대시보드 자동 업데이트 (Grafana 혹은 Fastify endpoint)

### 목표 2: False report 정확도 향상
- `false_report_count` 기록 방식 검토, 중복/오류 방지 로직 삽입

- 기존 34건 로그 분석 자동 스크립트 작성 및 정기 리포트
### 목표 3: Obsidian 문서·노트 관리 자동화
- `obsidian-watcher.ts`에 새 파일/변경 감지 후 메타데이터 자동 라벨링
- 개선 노트(`improvement_notes`) 자동 집계 스크립트 (`obsidian-context-builder.sh` 확장) 
- 주기적 정리 작업 (중복/오래된 문서 삭제) 스케줄링 (cron) 

## 자동화 가능한 부분

- **False report 검증**: 사전 스키마 검증, CI 단계에서 검사 
- **Obsidian 메타 관리**: 파일 감시 → 메타 자동 삽입 → 정리 스크립트
