# NCO 자가 발전 시스템 개선 전략

## 현재 상태 진단
- 전체 태스크: 100
- 완료된 태스크: 65
- 실패한 태스크: 35
- 멈춘 태스크: 0
- 성공률: 65.0%
- False report 횟수: 290
- Obsidian 문서 수: 2,480
- 개선 노트 수: 452

## 핵심 개선 목표 (3가지)
1. **태스크 성공률 75% 이상 달성** – 현재 65%에서 10%p 상승 목표.
2. **실패 태스크 원인 자동 탐지** – 실패 원인 분석 자동화 비중 80% 이상.
3. **False report 최소화** – 연간 false report 200건 이하로 감소.

## 구체적 실행 계획
### 목표 1: 성공률 75% 이상
- 기존 실패 원인 데이터베이스 구축
- 성공/실패 예측 모델 적용 및 프라임 태스크 우선순위 재조정
- 주요 병목 구간에 대한 리소스 할당 자동 조정 (CPU/Redis) 구현

### 목표 2: 실패 원인 자동 탐지
- `src/core/eventBus.ts`에 실패 이벤트 캡처 로직 추가
- 실패 로그를 구조화된 JSON으로 저장하고, 매일 분석 파이프라인 실행
- 자동 알림 및 재시도 전략 적용 (exponential backoff)

### 목표 3: False report 최소화
- `src/security/CommandGate.ts` 강화 – 허용되지 않은 명령 검증 로깅 수준 상향
- False report 발생 시 자동 티켓 생성 및 담당자 알림
- 월간 보고서에 false report 추세 시각화 추가

## 자동화 가능한 부분
- **배포 파이프라인**: CI에서 새로운 성공률 목표를 검증하고 자동 배포 트리거
- **실패 원인 분석**: 매일 00:00에 `npm run analyze-failures` 스크립트 실행 → 결과를 `obsidian`에 기록
- **보고서 생성**: `npm run generate-metrics-report` 로 월간 메트릭스 마크다운 자동 생성
