# EDUCATION-POLICY.md v2.1 심화

## 1. opencode 품질 게이트 수치

- **최소 콘텐츠 크기**: 2 KB 이상
- **품질점수**: 3.5 / 5.0 이상
- **검증자 리뷰**: 최소 3건 이상
- **수치 근거**:
  - 콘텐츠 크기는 팀 평균 문서 크기(1.8 KB)보다 10 % 이상 큰 것이 충분한 상세도를 보장한다.
  - 품질점수는 자체 평가 매트릭스(가독성, 정확성, 재현성)에서 70 % 이상 만족하도록 설계되었다.
  - 검증자 수는 다양한 관점을 확보하기 위해 최소 3명으로 제한한다.
- **예외 조건**:
  - 급박한 버그 수정이나 긴급 보안 패치의 경우, 최소 콘텐츠 크기와 검증자 수를 1명으로 낮출 수 있다.
  - 파일이 자동 생성되는 경우(예: 코드 생성 스크립트)에는 품질점수와 검증자 리뷰를 면제한다.

## 2. gemini 기여 등급 보상 정밀화

| 등급      | NVC 포인트 | 비고 |
|-----------|------------|------|
| Bronze    | +10        | - |
| Silver    | +30        | - |
| Gold      | +70        | - |
| Platinum  | +150       | - |

- **월 상한**: 각 사용자는 월 최대 **300 NVC** 까지 획득 가능 (예외: 특별 프로젝트 보상은 별도 적용).
- **멘토 보상 중복 여부**:
  - 멘토가 동시에 기여자와 멘티 역할을 수행할 경우, 멘토 보상은 **한 번만 적용**된다.
  - 멘티 보상과는 별도로 누적된다.

## 3. codex VC 발급 조건

- **완료율 기준**: 70 % 이상
  - **측정 방법**: `(제출된 작업 수 / 검증된 작업 수) × 100%`
  - 제출은 실제 PR 혹은 작업 티켓 생성, 검증은 `ollama` 검증 혹은 검증자 리뷰 완료를 의미한다.
- **VC 유효기간**: 발급일로부터 **6개월** 후 자동 소멸.
- **부정 발급 탐지 메커니즘**:
  - 동일 IP/계정에서 7일 내 3회 이상 VC 발급 요청 시 자동 플래그.
  - 작업 제출‑검증 비율이 30 % 이하인 경우 의심 검토 대상.
  - 비정상적인 보상 패턴 감지를 위해 `codex` 로그와 `ollama` 검증 결과를 교차 검증한다.
