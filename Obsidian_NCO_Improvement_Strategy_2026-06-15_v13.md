# NCO 자가 발전 시스템 개선 전략

## 현재 상태 진단
- 총 작업: 100
- 완료: 66 (성공률 66%)
- 실패: 31
- 정체(스턱): 3
- 거짓 보고 건수: 287
- Obsidian 문서 수: 2,433
- 개선 노트 수: 448

## 핵심 개선 목표 (3가지)
1. **성공률 80% 이상 달성** – 실패 및 정체 작업 감소.
2. **거짓 보고 0에 가깝게 최소화** – 보고 정확성 및 검증 체계 강화.
3. **지식 베이스(Obsidian) 활용 최적화** – 개선 노트 생산성 및 재사용성 향상.

## 구체적 실행 계획
### 목표 1: 성공률 80% 달성
- 실패 작업 원인 분석 자동화 (실패 로그 수집 → 패턴 매칭).
- 정체 작업 자동 재시도 스케줄러 도입 (exponential backoff).
- 주요 병목 단계 별 타임아웃/리트라이 정책 강화.
- 테스트 커버리지 90% 이상 유지 (CI 파이프라인에 통합).

### 목표 2: 거짓 보고 최소화
- 보고 전 T1 검증 추가: 파일 존재·내용 검증, DB 레코드 일관성 확인.
- 보고 API 응답에 검증 단계 메타데이터 포함.
- 거짓 보고 감지 알림 시스템 도입 (Slack/메일).
- 거짓 보고 발생 시 자동 롤백 및 재실행 워크플로우.

### 목표 3: Obsidian 활용 최적화
- 개선 노트 템플릿 자동 생성 스크립트 작성 (`npm run gen-improvement-note`).
- 기존 노트 메타데이터 (작성일, 담당자, 관련 지표) 표준화.
- 노트와 코드베이스 사이 링크 자동 생성 (코드 파일 ↔️ 노트).
- 주간 자동 요약 보고서 생성 (LLM 요약 활용).

## 자동화 가능한 부분
- **실패/정체 작업 재시도**: cron+worker 스크립트.
- **거짓 보고 검증**: pre‑commit hook / API 미들웨어.
- **노트 템플릿 및 메타데이터**: `scripts/generate_note.ts`.
- **주간 지표 대시보드**: GitHub Actions → Markdown → Obsidian sync.
