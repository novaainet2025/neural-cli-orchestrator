## 핵심 개선 목표 (3가지)
1. **스틱 태스크 최소화** – 자동 탐지·재시도·알림으로 작업 정체 해소
2. **보고 정확도 향상** – False report 검증·자동 교정 파이프라인 구축
3. **지식 베이스 효율화** – Obsidian 문서와 개선 노트 동기화·메타데이터 자동 생성

## 구체적 실행 계획 (각 목표별)
### 1. 스틱 태스크 최소화
- `tasks_stuck` > 0 시 Redis 이벤트 `task:stuck` 발행
- 워커 구현: 30초 간격으로 스틱 태스크 상태 조회, 자동 재시도 혹은 알림
- 모니터링 대시보드에 스틱 태스크 실시간 차트 추가
- 재시도 한계 초과 시 관리 콘솔에 티켓 생성

### 2. 보고 정확도 향상
- `false_report_detail` 스키마 확장 및 로그 수집
- 매 사이클 종료 시 `ollama` 검증 단계에서 보고서와 실제 결과 비교 자동화
- 차이 발견 시 자동 교정 스크립트 실행 및 관리자 알림
- 월간 보고서에 False report 비율 KPI 포함

### 3. 지식 베이스 효율화
- Obsidian API 이용해 문서와 개선 노트 메타데이터 매핑 자동화
- 신규/수정 문서 발생 시 자동으로 개선 노트 템플릿 생성
- 오래된 노트(90일 이상) 자동 아카이브 및 요약 생성
- 문서·노트 비율 목표: 1% 이하 유지

## 자동화 가능한 부분
- 스틱 태스크 탐지·재시도 워커 (Redis + Node worker)
- False report 검증 파이프라인 (ollama + 자동 교정 스크립트)
- Obsidian 문서·노트 동기화 스크립트 (Obsidian REST API 또는 파일 시스템 감시)
- KPI 대시보드 업데이트 자동화 (Grafana + Prometheus exporter)
