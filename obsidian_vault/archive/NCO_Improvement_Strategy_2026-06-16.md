## 현재 상태 진단
- 전체 태스크 100개 중 99개 완료, 1개 정체(stuck) → 성공률 99%
- 실패 태스크 0개, 오류 없음
- `false_report_count`=8 (잘못된 성공 보고 여부 확인 필요)
- Obsidian 문서 5165개, 개선 노트 711개 → 문서 대비 개선 비율 ~13.8%

## 핵심 개선 목표 (3가지)
1. **정체된 태스크 자동 탐지 및 해결**
2. **False Report 감소 및 검증 체계 강화**
3. **문서·노트 활용도 향상 및 자동화**

## 구체적 실행 계획
### 목표 1: 정체된 태스크 자동 탐지 및 해결
- 매 사이클 시작 시 `tasks_stuck` > 0 여부를 쿼리
- 정체 원인(LOCK, 외부 API 지연 등) 로그 수집
- 자동 재시도 워커 추가 (Redis 기반 백오프 전략)
- 재시도 3회 초과 시 알림 및 수동 검토 티켓 생성

### 목표 2: False Report 감소 및 검증 체계 강화
- `false_report_count` 메트릭을 T1 근거(파일/DB 실제 값)로 검증
- 각 성공 보고 시 SHA 해시와 결과 파일 체크섬 기록
- 검증 단계 실패 시 재실행 및 `false_report_count` 감소 로직 적용
- 월간 리포트에 감소 추이 시각화

### 목표 3: 문서·노트 활용도 향상 및 자동화
- `obsidian_docs` → 메타 데이터(태그, 연결) 자동 추출 (rg 검색 스크립트)
- 개선 노트(`improvement_notes`)와 연계된 자동 PR/이슈 생성 템플릿
- 주간 자동 요약 생성 스크립트 (`nco_task opencode` 활용) 및 Obsidian에 삽입
- 오래된 노트(30일 미사용) 자동 아카이브 정책

## 자동화 가능한 부분
- 정체 태스크 감시 및 재시도 워커 (cron + Redis)
- 성공 보고 검증 파이프라인 (post‑action hook)
- 문서 메타 데이터 수집 및 개선 노트 연동 스크립트
- 주간/월간 메트릭 대시보드 자동 업데이트 (Fastify endpoint + Grafana)
