## 현재 상태 진단
- 전체 태스크 100건 중 100건 완료, 성공률 100%로 운영이 원활함.
- 정체된 태스크 없음, 실패도 0건으로 안정적인 실행 흐름 확인.
- 허위 보고(`false_report_count`)가 2건 존재, 신뢰성에 잠재적 위험.
- Obsidian Vault에 문서 10,040개, 개선 노트 980개가 누적돼 지식 관리 규모가 큼.

## 핵심 개선 목표 (3가지)
1. **허위 보고 방지 및 검증 체계 강화** – `false_report_count` 0으로 감소시키고, 자동 검증 절차 도입.
2. **지식 관리 효율화** – 방대한 문서·개선 노트의 정리·중복 제거 및 활용도 향상.
3. **자동화 확대** – 기존 운영 흐름에서 자동화 가능한 부분을 식별·구현해 인적 개입 최소화.

## 구체적 실행 계획 (각 목표별)
### 목표 1: 허위 보고 방지 및 검증 체계 강화
- **검증 게이트**: 작업 완료 직후 T1 수준 파일·결과 존재 여부와 단위 테스트 자동 실행(`npm test --silent`).
- **교차 검증**: `cursor-agent`와 `ollama`를 활용한 결과 Spot‑Check을 병렬 수행.
- **패널티**: 허위 보고 발생 시 해당 에이전트 신뢰 점수 감소·다음 라우팅 시 가중치 부여.
- **모니터링**: 매 사이클 `false_report_count`를 대시보드에 KPI로 표시.

### 목표 2: 지식 관리 효율화
- **중복 탐지**: `gbrain` 검색엔진을 이용해 동일 문서·노트 해시 값 기반 dedup 스크립트(월 1회) 실행.
- **아카이브 정책**: 90일 이상 접근되지 않은 문서는 `archive` 폴더로 이동, 메타에 `archived_at` 타임스탬프 기록.
- **링크 정리**: 개선 노트와 해당 문서 간 양방향 링크 자동 생성(`[[DocID]]` 형태) 스크립트 구현.
- **인덱스 재구축**: `gbrain index --rebuild`를 주간 실행해 검색 정확도 유지.

### 목표 3: 자동화 확대
- **태스크 메타**: `lastUpdated` 타임스탬프와 `retryCount` 필드를 SQLite 마이그레이션으로 추가.
- **자동 재시도**: `tasks_stuck` 감지 시 `retryCount`가 3 이하이면 재시도, 초과 시 알림.
- **보고 자동화**: 매 사이클 종료 시 `npm run generate-report` 로 KPI 보고서 자동 생성·저장.
- **CI 연동**: GitHub Actions에 `npm test`와 `npm run lint` 단계 추가, 실패 시 자동 롤백 방지.

## 자동화 가능한 부분
- 허위 보고 검증(테스트·스팟체크) → CI 파이프라인.
- 중복 문서 탐지 및 아카이브 → 크론 잡(`gbrain dedup && mv ...`).
- 태스크 재시도 로직 → `src/core/taskManager.ts`에 재시도 핸들러 구현.
- KPI 대시보드 업데이트 → `src/server/monitor.ts`에 실시간 메트릭 플러그인 추가.
