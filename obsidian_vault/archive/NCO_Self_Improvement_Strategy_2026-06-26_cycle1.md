## 구체적 실행 계획 (각 목표별)
### 1. 작업 정체 방지 및 자동 복구
- **감지**: 30초 주기로 `tasks_stuck` 모니터링 (Redis 키 `task:stuck:*`).
- **복구**: 감지 시 해당 작업을 재큐에 넣고 최대 3회 재시도, 재시도 초과 시 알림.
- **책임/도구**: `cursor-agent`(감시 스크립트), `ollama`(재시도 검증).

### 2. 허위 보고 정확성 강화
- **검증 단계**: 모든 Type B/C 작업 완료 전 `npm test`·파일 존재·HTTP 응답 등 T1 증거 자동 확인.
- **보고 정책**: 검증 실패 시 `failed` 로 기록, `false_report_count` 증가, 해당 에이전트에 재시도 제한(3회).
- **대시보드**: 검증 통과율(Verified Success Rate) KPI 추가.
- **책임/도구**: `opencode`(검증 로직 설계), `codex`(구현), `ollama`(테스트 자동화).

### 3. 지식·노트 관리 자동화
- **중복 탐지**: 정기 `gbrain search` 로 중복 문서 탐색, 중복 비율 >30% 시 자동 태그 `duplicate`.
- **노후 아카이브**: 마지막 수정일이 180일 이상 된 노트는 `archived` 폴더로 이동.
- **TTL/점수**: 각 문서에 `freshness_score` 메타데이터 부여, 상위 50개만 RAG 컨텍스트 사용.
- **책임/도구**: `copilot`(스크립트 작성), `gbrain`(검색/점수), `cursor-agent`(자동 이동).
