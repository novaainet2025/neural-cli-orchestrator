## 구체적 실행 계획
### 목표 1 – 허위 보고 감소
- `false_report` 로그를 구조화된 DB 테이블에 저장하고, 원인(에이전트, 단계, 오류 코드)별 집계 파이프라인 구축.
- 주간 리포트 자동 생성 (`/tmp/false_report_weekly.md`) 및 담당자에게 Slack 알림.
- 허위 보고 발생 조건에 대한 룰 엔진 구현 → 자동 차단 혹은 재시도 정책 적용.

### 목표 2 – 스루풋 및 병목 최소화
- 작업 진행 단계별 타임스탬프를 Redis 스트림에 기록, `tasks_stuck` 감지 로직 추가 (예: 5분 이상 진행되지 않으면 알림).
- 워커 프로세스 자동 스케일링: CPU/메모리 사용량에 따라 워커 인스턴스 수 동적 조정.
- 멈춘 작업에 대한 자동 재시도 + 히스토리 저장, 재시도 횟수 초과 시 수동 검토 티켓 생성.

### 목표 3 – 자동화·모니터링 강화
- Prometheus exporter 추가 (`src/metrics/collector.ts`) → `tasks_total`, `tasks_completed`, `tasks_failed`, `tasks_stuck`, `false_report_count` 노출.
- Grafana 대시보드 템플릿 제공 (`obsidian_vault/improvement_notes/grafana_dashboard_template.json`).
- CI 파이프라인에 메트릭 검증 스텝 포함, 임계치 초과 시 빌드 실패.

## 자동화 가능한 부분
- **허위 보고 집계**: 매일 00:00에 스크립트 실행 → DB INSERT → 리포트 생성.
- **스루풋 감시**: Redis 스트림 구독 + Lambda‑like 함수가 5분 이상 정체 감지 시 자동 알림.
- **메트릭 수집**: `npm run metrics` 커맨드로 Prometheus 포맷 출력, Grafana 자동 스크레이핑.
- **재시도 로직**: 실패 시 `retry_policy` JSON 정의에 따라 exponential backoff 적용 자동화.
