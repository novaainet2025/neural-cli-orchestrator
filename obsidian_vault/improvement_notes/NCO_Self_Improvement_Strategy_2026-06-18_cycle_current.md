## 구체적 실행 계획 (각 목표별)
### 1. 작업 정체 해소 및 성공률 향상
- **원인 분석**: `tasks_stuck` 원인 로그 수집 파이프라인 구축 (`src/core/eventBus.ts`에 스테이터스 이벤트 추가).
- **재시도 메커니즘**: 실패/정체된 작업에 대해 자동 재시도 로직 구현 (exponential backoff 적용).
- **모니터링 대시보드**: WebSocket 브리지에 실시간 작업 상태 위젯 추가 (`src/server/monitor.ts`).
### 2. 거짓 보고 검증 체계 강화
- **검증 레이어**: 보고 전후에 검증 스텝 삽입 (`src/security/CommandGate.ts`에 검증 훅 추가).
- **리스크 점수**: 거짓 보고 횟수 기반 에이전트 신뢰 점수 모델링, 일정 임계값 초과 시 알림.
- **데이터 정합성 검사**: SQLite ↔ Redis 동기화 시 정합성 체크 스크립트 (`scripts/verify-sync.sh`).
### 3. 지식·노트 관리 자동화
- **메타데이터 스키마**: Obsidian 파일 헤더에 `tags`, `category`, `last_updated` 추가.
- **인덱스 빌더**: `scripts/build-obsidian-index.js`로 전체 문서 인덱스 생성 및 검색 가능 DB (SQLite) 저장.
- **정기 정리**: 매 사이클 종료 시 중복·오래된 노트 자동 아카이브 (`npm run prune-obsi`).

## 자동화 가능한 부분
- **작업 재시도**: 자동 백오프 재시도 스케줄러 (cron).
- **거짓 보고 검증**: 보고 단계별 자동 검증 스크립트와 알림 (`src/security/validator.ts`).
- **노트 인덱싱**: CI 파이프라인에 인덱스 빌드 단계 추가, PR 자동 실행.
- **정합성 검사**: `npm run verify-sync`를 CI 후크에 포함.
