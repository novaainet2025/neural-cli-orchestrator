## 핵심 개선 목표 (3가지)
1. **작업 성공률 향상** – 현재 63 %에서 80 % 이상으로 끌어올림.
2. **작업 정체 감소** – 멈춰 있는 작업 수를 0으로 감소.
3. **거짓 보고 정확도 개선** – false_report_count 를 100 이하로 감소.

## 구체적 실행 계획 (각 목표별)
### 1️⃣ 작업 성공률 향상
- **원인 분석**: 실패 작업 로그 자동 수집 및 라벨링 파이프라인 구축.
- **핵심 개선**: 고빈도 실패 원인(리소스 부족, 타임아웃, 외부 API 오류) 별 자동 재시도 및 백오프 전략 적용.
- **측정 지표**: 매일 성공률, 재시도 성공 비율, 평균 작업 소요 시간.

### 2️⃣ 작업 정체 감소
- **감시 체계**: `tasks_stuck` 감지 알림을 Redis 기반 이벤트로 전파하고, 5분 이상 정체 시 자동 재배치.
- **스케줄러 개선**: 작업 우선순위 재조정 및 백그라운드 워커 풀 확장.
- **측정 지표**: 멈춰 있는 작업 평균 지속시간, 재시도 후 정상 복귀 비율.

### 3️⃣ 거짓 보고 정확도 개선
- **보고 검증**: false_report_count 를 발생 원천별(모니터링, 휴먼 오류) 로 분류하고, 자동 교정 로직 적용.
- **피드백 루프**: 보고된 오류를 UI 에 바로 표시하고, 담당자에게 즉시 알림.
- **측정 지표**: 월별 false_report_count 감소 추이, 자동 교정 적용 건수.

## 자동화 가능한 부분
- **메트릭 수집 자동화**: `src/metrics/collector.ts` 에서 현재 성능 지표를 1분 주기로 DB 및 Redis 에 저장.
- **보고서 생성 파이프라인**: 새로운 메트릭이 저장될 때마다 위 마크다운 템플릿을 기반으로 자동 업데이트 스크립트 실행 (`npm run generate-improvement-report`).
- **알림 및 재시도**: `src/agent/TaskScheduler.ts` 에서 `tasks_stuck` 및 `tasks_failed` 를 감지하면 즉시 Slack/Webhook 로 알림 및 재시도 로직 트리거.
