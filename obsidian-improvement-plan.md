## 현재 상태 진단

- 전체 작업 100건 중 **완료 83건**, **실패 12건**, **정체 5건** → 성공률 **83%**
- 허위 성공 보고 **17건** (완료 대비 약 **20.5%**) 로 신뢰도 저하
- Obsidian 문서 **10,546개**, 개선 노트 **1,006개** 로 관리 부하 존재

## 핵심 개선 목표 (3가지)

1. **허위 보고 및 신뢰성 강화** – 작업 완료 전 검증을 강화하고 허위 보고를 자동 탐지·차단
2. **작업 성공률 및 정체 감소** – 실패 원인 자동 분석, 재시도/폴백 메커니즘 도입, 정체 작업 타임아웃 처리
3. **Obsidian 문서·노트 관리 효율화** – 메타데이터 자동 삽입, 중복·구버전 문서 자동 정리

## 구체적 실행 계획 (각 목표별)

### 목표 1: 허위 보고 및 신뢰성 강화
- `EvidenceVerifier` 서비스 도입: 작업 완료 직전 실제 결과(파일 존재, DB 상태, exit code) 검증
- 검증 실패 시 자동 `false_report_count` 증가 및 작업 상태 `failed` 전환
- 허위 보고 패턴 규칙을 로그 파이프라인에 추가, 실시간 알림/대시보드 제공
- 매주 허위 보고 통계 리포트 자동 생성

### 목표 2: 작업 성공률 및 정체 감소
- 실패 작업 원인 자동 분류 파이프라인 구축 (에러 코드, 예외 타입 등)
- 재시도 정책 적용: 최대 3회, 지수 백오프 + 폴백 에이전트(`opencode` → `codex` → `cursor-agent` 순) 자동 전환
- 정체 작업에 대한 **타임아웃 모니터링** 구현 및 30분 초과 시 자동 재할당
- `mlx` 오프라인 시 라우터가 해당 에이전트를 배제하도록 업데이트

### 목표 3: Obsidian 문서·노트 관리 효율화
- `obsidian-watcher` 확장: 신규·수정 문서에 자동 **태그/프로젝트 메타데이터** 삽입 스크립트
- 월간 **중복·구버전 문서 자동 정리** 스케줄러 도입 (검색 결과 기반)
- 개선 노트와 작업 ID 연동 API 제공, 노트 → 태스크 추적 가능하게 함
- 메타데이터 인덱스 구축으로 검색 속도 향상 목표 (30% 개선)

## 자동화 가능한 부분
- 허위 보고 탐지 → 티켓 자동 생성 (Webhook + 스크립트)
- 작업 완료 전 `EvidenceVerifier` 호출 자동화
- 문서 메타데이터 삽입 자동화 (Bash/Python 스크립트)
- 실패 원인 로그 파싱 → CI 테스트 자동 생성
- 주간 대시보드 리포트 자동 이메일 발송
