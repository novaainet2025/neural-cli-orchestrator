## 구체적 실행 계획

### 목표 1: 허위 완료 보고 감소
- **Stop 훅·품질 게이트** 도입 (`nco-stop-quality-gate.sh`).
- 완료 전 `tsc --noEmit`와 전체 테스트(`npm run test:run`) 실행, 실패 시 자동 `failed` 전환.
- 허위 보고 감지 시 `false_report_count` 증가, 해당 작업을 `failed` 로 재오픈하고 알림 전송.
- 모니터링: 매일 `false_report_count` 로그와 실시간 대시보드에 표시.

### 목표 2: 개선 노트 적체 해소
- **주간 배치 파이프라인** 구축: `improvement_notes` 를 자동으로 `nco_task` 로 변환。
- 중복·우선순위 알고리즘 적용 후 `Commander` 로 자동 태스크 생성。
- 30일 이상 미처리 노트는 자동 아카이브·폐기 규칙 적용。
- KPI: 주간 처리량 ≥ 80% (≈ 800 노트/주)。

### 목표 3: Obsidian 지식 과부하 최적화
- **시맨틱 인덱스** (`gbrain` 혹은 `bm25`) 구축, 문서 메타데이터에 TTL 부여。
- 핫(최근 30일)·콜드(그 외) 분리，핫 문서만 에이전트 컨텍스트에 주입。
- 검색 결과 상위 N%만 직접 전달，나머지는 요약·링크 형태로 제공。
- 정기 리뷰: 매월 인덱스 재생성 및 사용량 분석。
