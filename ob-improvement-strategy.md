## 현재 상태 진단

- 전체 작업 100건 중 81건 완료, 14건 실패, 0건 정체, 성공률 81%
- 허위 보고 14건 포함, 실질 성공률 약 67% ((81-14)/100)
- Obsidian 문서 10,350개, 개선 노트 992개로 문서·노트 관리 부하가 높음

## 핵심 개선 목표 (3가지)

1. **허위 완료 보고 감소 및 신뢰도 개선** – 허위 보고 자동 감지, 사전 검증 강화
2. **실패 작업 원인 분석 및 자동 복구** – 실패 원인 로그 수집·분석 파이프라인 구축
3. **문서·노트 관리 효율화** – 메타데이터 자동 삽입, 중복 정리, 관찰 자동화

## 구체적 실행 계획 (각 목표별)

### 목표 1: 허위 완료 보고 감소 및 신뢰도 개선
- `nco-stop-quality-gate.sh` 후크에 TypeScript 컴파일 및 테스트 자동 실행 추가
- 작업 완료 전 `EvidenceVerifier` 서비스 호출, 검증 실패 시 `false_report_count` 증가 및 상태 `failed` 전환
- 허위 보고 패턴 규칙을 로그 파이프라인에 추가, 주간 대시보드와 알림 제공

### 목표 2: 실패 작업 원인 분석 및 자동 복구
- 각 에이전트 실행 로그에 `taskId`와 `errorCode` 메타데이터 기록
- `failure-analyzer` 마이크로서비스 구축, 실패 원인(타임아웃, 의존성 오류 등) 자동 분류
- 재시도 파이프라인 구현 – 자동 복구 가능한 경우 재시도, 재시도 실패 시 인간 검토 티켓 생성

### 목표 3: 문서·노트 관리 효율화
- Obsidian 문서 생성/수정 시 메타데이터(태그, 생성일, 관련 taskId) 자동 삽입 스크립트 (`obsidian-context-builder.sh` 개선)
- 중복 문서·노트 탐지 스크립트 추가, 주간 청소 작업 자동화
- 개선 노트와 실제 작업 연결 자동화 – `improvement_notes` DB와 Obsidian 문서 매핑

## 자동화 가능한 부분
- **품질 게이트**: `nco-stop-quality-gate.sh` 후크 자동 실행 (tsc, vitest)
- **로그 파이프라인**: `obsidian-watcher.ts`에 허위 보고/실패 패턴 필터링 추가
- **대시보드**: Grafana/Prometheus와 연동한 실시간 KPI 제공
- **재시도 파이프라인**: 실패 작업 자동 재시도 및 티켓 생성 자동화
