## 구체적 실행 계획
### 목표 1: 정체 작업 감소
- 원인 분석: 이벤트 버스 재시도 로직, Redis 연결 불안정
- 구현 단계:
  - (a) 이벤트 재시도 백오프 전략 적용 (exponential backoff)
  - (b) 정체 작업 자동 재시도 워커 추가 (cron 5분)
  - (c) 정체 감지 알림 → Slack/Webhook 연동
- 측정 지표: 정체 작업 수 2→0, 평균 복구 시간 <30s

### 목표 2: false report 정확도 향상
- 현재 false_report_count가 문자열 "1" 로 저장되어 타입 불일치 위험
- 구현 단계:
  - (a) DB 스키마 수정: `false_report_count`를 INTEGER 로 마이그레이션
  - (b) 보고 API 검증 레이어 추가 (Zod 스키마)
  - (c) 자동 감시 로그 분석 도구 도입 (ELK)
- 측정 지표: false report 오탐률 <5%, 보고 지연 <2s

### 목표 3: Obsidian 연동 자동화

  - (b) 메타데이터(태그, 상태) 자동 추출 및 DB 반영
  - (c) 주기적 요약 리포트 생성 (Markdown 템플릿)
