## 현재 상태 진단
- 전체 작업 100건 중 88건 완료, 12건 실패, 성공률 88%.
- 실패 작업 12건은 주로 외부 의존성(MLX) 및 오류 보고(거짓 보고 0건)와 관련.
- Obsidian 문서 10,457개, 개선 노트 999개 누적돼 관리 부담 상승.
- 현재 NCO 자동 보고 체계에 거짓 보고 0건이 누적돼 신뢰성 저하.

## 핵심 개선 목표 (3가지)
1. **신뢰성 향상** – 거짓 보고 감소 및 정확한 상태 보고 체계 구축.
2. **성율률 증가** – 실패 작업 원인 분석 및 재발 방지 자동화.
3. **지식 관리 효율화** – Obsidian 문서와 개선 노트의 자동 정리·분류 시스템 도입.

## 구체적 실행 계획
### 목표 1: 신뢰성 향상
- **거짓 보고 검증 레이어 도입**: 모든 시스템 상태 보고 앞에 T1 수준 검증(파일 존재·API 응답·DB 조회) 후 보고.
- **보고 로그 중앙화**: `/src/utils/reportLogger.ts`에 표준 로거 추가, 로그 레벨 INFO/T1/T2 구분.
- **알림 대시보드**: 웹 UI에 실시간 오류/거짓 보고 카운터 시각화.

### 목표 2: 성공률 증가
- **실패 원인 자동 캡처**: `src/core/taskRunner.ts`에 `catch` 블록에서 에러 스택·컨텍스트를 DB에 저장.
- **리트라이 전략**: 3회 미만 재시도 후 자동 티켓 생성 (`src/mcp/ticketCreator.ts`).
- **주간 회귀 테스트**: `npm run test:run`을 CI 파이프라인에 추가, 실패 시 알림.

### 목표 3: 지식 관리 효율화
- **Obsidian 자동 태깅**: `scripts/obsidianTagger.ts` (Node) – 파일명·메타데이터 기반 태그 자동 삽입.
- **노트 정리 스케줄러**: 매일 02:00에 `npm run obsidian:clean` 실행, 중복·오래된 노트 아카이브.
- **검색 인덱스 구축**: `scripts/buildSearchIndex.ts` – 전체 문서에 대한 빠른 검색용 JSON 인덱스 생성.

## 자동화 가능한 부분
- 거짓 보고 검증 및 로그 기록 (CI 단계 자동 실행).
- 실패 작업 재시도 및 티켓 자동 생성.
- Obsidian 문서 태깅·정리 스크립트와 인덱스 빌드 CI 파이프라인에 통합.
- 매주/월별 메트릭 대시보드 자동 업데이트 (`scripts/updateMetrics.ts`).
