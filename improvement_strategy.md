## 현재 상태 진단
- 전체 작업 100건 중 75건 완료 (표면 성공률 75%).
- 허위 완료 15건 존재하여 실제 성공률은 **60%**.
- 실패 17건, 정체 3건, 잔여(진행·대기) 5건.
- 허위 완료 비율 **20%** 로 품질 검증 부족을 시사.
- 현재 에이전트 가동 상황: `cursor-agent`만 작동, `mlx` 오프라인 → 처리 용량·복구력 제한.
- Obsidian vault에 문서 10,393개, 개선 노트 995개가 축적돼 있어 관리·검색 부담 존재.

## 핵심 개선 목표 (3가지)
1. **허위 완료 감소 및 실질 성공률 향상** – 검증 게이트 도입으로 실제 완료 여부를 확인.
2. **에이전트 다중화 및 복원력 강화** – `mlx` 복구 및 추가 에이전트 도입으로 단일 실패점 제거.
3. **지식 베이스 자동화 및 품질 관리** – Obsidian 문서 자동 인덱싱·중복 제거·노트 품질 검토 파이프라인 구축.

## 구체적 실행 계획 (각 목표별)
### 1. 허위 완료 감소
- **완료 전 검증 단계** 구현
  - 작업 완료 직전 `tsc` 빌드와 단위·통합 테스트 실행.
  - API 응답 검증 (`curl localhost:6200/...` 결과와 기대값 비교).
  - 결과 파일 존재·해시 검증.
- **자동 롤백**: 검증 실패 시 상태를 `failed` 로 전환하고 재할당.
- **보고서**: 매일 `false_report_count` 를 로그와 DB에 기록, 알림 트리거.

### 2. 에이전트 다중화
- **mlx 복구**: 최신 Docker 이미지 배포·헬스체크 스크립트 추가 (`healthcheck.sh`).
- **보조 에이전트 추가**: `codex`와 `openrouter`를 보조 에이전트로 등록, 라우팅 로직에 `fallback` 정책 적용.
- **에이전트 헬스 모니터링**: 30초 간격 `curl localhost:6201/health` 확인, 비정상 시 자동 재시작 (`pm2 restart`).
- **로드 밸런싱**: 작업 큐를 Redis 스트림에 분산, 에이전트별 가중치 설정.

### 3. 지식 베이스 자동화
- **Obsidian 인덱싱 파이프라인**
  - `gbrain` 또는 `ripgrep` 기반 주기적 인덱스 생성 (`watch.sh`), 변동 감지 시 자동 업데이트.
- **중복·불필요 노트 정리**
  - 스크립트 `detect-duplicates.js` 로 해시 기반 중복 탐지, 리뷰 후 자동 삭제/병합.
- **노트 품질 체크**
  - 메타데이터 (`tags`, `last_modified`) 검증, 오래된·미사용 노트에 리마인드 알림.
- **CI 연동**: `npm run test:docs` 로 문서 빌드·링크 검증, 실패 시 PR 자동 생성.

## 자동화 가능한 부분
- **검증 게이트**: CI 파이프라인에 `npm run verify-task` 스크립트 추가 → 자동 실행.
- **에이전트 헬스 체크**: `pm2` 에코시스템 파일에 `watch:true` 설정, 실패 시 자동 재시작.
- **Obsidian 인덱스·정리**: GitHub Actions 혹은 로컬 cron(5분)으로 `gbrain index` 및 `detect-duplicates.js` 실행.
- **보고서 자동 생성**: 매일 `npm run daily-report` 로 JSON → 마크다운 변환, Obsidian `daily/` 폴더에 커밋.
