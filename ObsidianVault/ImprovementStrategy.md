# NCO 자가 발전 시스템 개선 전략

## 현재 상태 진단
- 전체 작업 100건 중 90건 완료, 성공률 90% (목표 95% 미달)
- 실패 5건, 정체 5건 → 정체율 5%, 실패율 5%
- 허위 완료 보고 2건 → 실제 성공률 약 88%
- Obsidian 문서 9,693건, 작업당 평균 96.5개 문서 생성 → 문서 과다 생성 및 관리 부하
- 개선 노트 971건 → 문서 대비 약 1:10 비율, 계획·실행 불균형

## 핵심 개선 목표 (3가지)
1. **작업 정체 및 실패 감소** – 정체율을 2% 이하, 실패율을 2% 이하로 낮춤
2. **KPI 신뢰성 강화** – 허위 완료 보고를 0건으로 줄이고 실질 성공률을 95% 이상 달성
3. **문서·노트 효율화** – 작업당 문서 생성량 30% 감축 및 개선 노트 활용도 80% 이상 향상

## 구체적 실행 계획
### 목표 1: 작업 정체 및 실패 감소
- **모니터링**: 5분 이상 응답 없는 작업 자동 감지 로직 추가 (Redis TTL 기반).
- **자동 재시도**: 감지된 작업을 지수 백오프(1, 2, 4분) 후 3회 재큐, 재시도 실패 시 `failed` 상태 전환.
- **원인 로깅**: 파일 락, 에이전트 hang, 도구 블록 등 원인 자동 기록 및 알림 (Slack/webhook).
- **테스트**: 시뮬레이션 테스트 추가 (`tests/stuck-task.test.ts`).

### 목표 2: KPI 신뢰성 강화
- **완료 검증**: `tasks_completed` 증가 시 결과 검증 플러그인 실행 – DB 레코드와 실제 산출물(파일, DB, 외부 API) 검증.
- **허위 보고 방지**: `false_report_count`를 0으로 초기화하고, 허위 보고 발생 시 자동 롤백 및 알림.
- **대시보드**: 실시간 KPI 대시보드에 `실질 성공률`(완료‑허위)/전체 표시.

### 목표 3: 문서·노트 효율화
- **템플릿 도입**: 작업당 기본 템플릿 제공, 불필요한 자동 문서 생성을 방지.
- **자동 정리**: 30일 이상 미사용 문서 자동 아카이브 스크립트 (`scripts/archive-old-docs.ts`).
- **노트 활용**: 개선 노트를 주간 스프린트 회고에 자동 집계, 미완료 노트에 리마인더 연결.

## 자동화 가능한 부분
- **정체 탐지 & 재시도**: Redis TTL + cron job 자동 실행.
- **완료 검증 파이프라인**: CI 단계에 검증 스크립트 삽입.
- **문서 아카이브**: GitHub Actions 정기 실행.
- **대시보드 업데이트**: WebSocket 이벤트 기반 실시간 반영.

## 구체적 실행 계획 (각 목표별)
### 1. 성공률 향상
- **원인 분석**: 최근 7일간 실패 작업 로그 자동 수집 및 분류 스크립트 구현
- **재시도 로직**: `TaskRunner`에 지수 백오프 재시도(최대 3회) 도입
- **리소스 최적화**: Redis‑SQLite 동기화 간격 축소 및 rate‑limit 관리 UI 추가
- **테스트 강화**: CI에 회귀 테스트 및 실패 시 자동 알림 연동

### 2. 거짓 보고 방지
- **검증 영수증 표준화**: 모든 작업 종료 시 T1 수준 증거(파일, DB, HTTP) 자동 기록
- **자동 검증 파이프라인**: `nco_task ollama` 로 결과물 검증 스크립트 실행
- **모니터링**: 거짓 보고 카운트를 실시간 대시보드에 노출, 임계치 초과 시 슬랙 알림

### 3. 자동화 확대
- **작업 템플릿화**: 공통 플러그인/스크립트 레포 구축(`src/utils/*`)
- **CI/CD 파이프라인**: GitHub Actions에 `npm run test`·`npm run build` 자동 실행 및 성공 시 PR 자동 생성
- **Obsidian 연동**: `obsidian-sync.ts` 스크립트로 개선 노트를 자동 커밋·동기화
- **대시보드 실시간 업데이트**: Redis Pub/Sub → WebSocket → Grafana 연동

## 자동화 가능한 부분
