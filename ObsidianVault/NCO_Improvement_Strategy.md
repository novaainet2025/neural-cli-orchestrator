## 구체적 실행 계획

### 목표 1: 허위 완료 보고 감소
- **검증 파이프라인 구축**: 작업 최종 단계에 `file_exists`, `test_pass`, `shell_success` 등 최소 2개의 실체 검증을 mandatory 로 설정
- **에이전트 신뢰 점수 도입**: 허위 보고 시 해당 에이전트 신뢰 점수 -10, 연속 실패 시 자동 재시도 제한
- **대시보드 KPI**: `검증 통과율`(전체 작업 중 검증 성공 비율)과 `허위 보고율`(전체 작업 대비 허위 보고 비율) 추가
- **알림**: 허위 보고 발생 시 Slack/Webhook 으로 실시간 알림

### 목표 2: 백로그 실행률 향상
- **자동 트리거**: `improvement_notes`에 신규 항목이 생성될 때마다 CI 파이프라인에 JIRA/Trello 티켓 자동 생성
- **우선순위 알고리즘**: 영향도·긴급도·실행 난이도 기반 스코어링 → 상위 20%를 매 사이클 자동 할당
- **주간 스프린트 리뷰**: `cursor-agent`가 진행 중인 개선 노트를 요약하고 담당 에이전트에 할당

### 목표 3: 지식베이스 최적화
- **중복 탐지 스크립트**: `rg` 로 유사 문서 제목·내용 검색, 중복 80% 이상인 경우 자동 병합 제안
- **메타데이터 강화**: 각 문서에 `tags`, `last_reviewed`, `relevance_score` 메타 추가
- **검색 엔진 튜닝**: Obsidian 플러그인 `obsidian-search-plus` 설정 조정, 인덱스 재생성 주기 1주일로 설정
