## 핵심 개선 목표 (3가지)
1. **허위완료 보고 감소** – FalseReportGuard 강화 및 검증 프로세스 도입
2. **작업 정체 및 실패 감소** – Heartbeat·타임아웃 자동 감지 및 재시도 정책 적용
3. **개선 노트 실행률 향상** – 노트 자동 과제화 및 실행 트래킹 시스템 구축

## 구체적 실행 계획 (각 목표별)
### 목표 1: 허위완료 보고 감소
- **검증 레이어 추가**: 작업 완료 전 `FalseReportGuard`가 파일 존재, 테스트 통과, exit code 0 등을 확인하도록 구현
- **KPI 모니터링**: 에이전트별 허위완료 비율을 실시간 대시보드에 기록
- **Stop Hook 강화**: 허위완료가 감지되면 작업 중단 및 알림 전송

### 목표 2: 작업 정체 및 실패 감소
- **Heartbeat 개선**: 5분 간격 heartbeat 누락 시 자동 재할당 로직 추가
- **타임아웃 재시도**: 30분 무응답 시 작업 재시도 및 에스컬레이션 정책 수립
- **실패 원인 분류**: `tasks_failed` 원인을 타임아웃/검증/에이전트 별로 라벨링하고 재시도 전략 적용

### 목표 3: 개선 노트 실행률 향상
- **노트 자동 과제화**: `improvement_notes` 중 심각도·중복도 기준 상위 N건을 자동으로 `tasks` 테이블에 삽입
- **실행 트래킹**: `verified_at` 필드 추가, 완료 시 자동 업데이트
- **주간 리포트**: 실행 비율과 미처리 노트 수를 주간 보고서로 생성

## 자동화 가능한 부분
- `FalseReportGuard`와 Heartbeat 감시 로직을 CI 파이프라인에서 자동 검증
- 개선 노트 → 작업 변환 스크립트를 정기(cron) 실행
- 성공/실패/정체 메트릭을 Grafana 대시보드에 실시간 시각화
