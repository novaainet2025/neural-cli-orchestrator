## 구체적 실행 계획
### 목표 1: 정체 작업 감소 및 자동 회복
- 작업 스케줄러에 타임아웃 감시 로직 추가
- 정체 감지 시 자동 재시도 및 알림 메커니즘 구현
- 재시도 횟수 초과 시 관리자에게 이슈 티켓 생성
### 목표 2: 잘못된 보고 정확도 향상
- `false_report_count` 집계 로직 검증 및 데이터 타입 정규화(`string`→`number`)
- 로그 레벨 및 필터링 강화, 중복/오류 보고 방지
- 정밀 검증 테스트 추가 (유닛·통합)
### 목표 3: 문서·노트 관리 자동화
- Obsidian Vault 자동 인덱싱 스크립트 구현 (새 파일 감시)
- 개선 노트 템플릿 자동 생성 및 메타데이터 삽입
- 주기적 정리 작업(CLI)으로 오래된/중복 노트 정리

## 자동화 가능한 부분
- 정체 작업 감시·재시도 → `src/core/monitor.ts`에 스케줄러 플러그인
- false report 집계 검증 파이프라인 → CI 테스트 파이프라인에 추가
- Obsidian 파일 감시 및 템플릿 적용 → `scripts/obsidian_sync.sh` 배포 및 cron 실행
