## 구체적 실행 계획 (각 목표별)
### 1️⃣ 거짓 보고 방지 및 검증 강화
- **로그 분석**: `false_report` 로그를 주간 집계, 패턴 분석 스크립트 작성.
- **T1 검증 로직**: 파일·DB 존재·내용 검증 후 리포트 생성 함수(`verifyReport.ts`) 추가.
- **자동 교정 워크플로**: 검증 실패 시 `codex`를 통해 리포트 자동 수정, `ollama`로 최종 검증.
- **알림**: 검증 실패 시 Slack/Discord 알림 연동.

### 2️⃣ 지식베이스 자동 정제·연결
- **메타데이터 추출**: `obsidian` 폴더 전체 파일을 `rg`로 스캔, YAML front‑matter 자동 삽입.
- **중복·낡은 문서 탐지**: 텍스트 유사도 기반(`nvidia` embed) 스크립트 구현, 정리 PR 자동 생성.
- **연결 자동화**: 새로운 노트가 생성될 때 관련 기존 노트와 하이퍼링크 자동 삽입.
- **CI 통합**: PR 검증 단계에 `gbrain` 검색 검증 추가.

### 3️⃣ 지속 가능한 메트릭 수집·피드백 루프 구축
- **대시보드**: Grafana 혹은 simple HTML 대시보드에 `tasks_total`, `tasks_completed`, `false_report_count`, `docs_count` 등 지표 시각화.
- **주기적 보고**: 매주 `npm run metrics` 스크립트 실행 → Markdown 보고서 자동 생성 → Obsidian에 저장.
- **피드백 루프**: 개선 노트(`improvement_notes`)와 실제 메트릭을 매 사이클 매핑, 목표 달성률 계산.

## 자동화 가능한 부분
- **보고 검증**: `verifyReport.ts` → CI 파이프라인에 후크 추가.
- **Obsidian 문서 메타데이터 관리**: `obsidian/meta_sync.ts` 스크립트 → Git hook 또는 CI 자동 실행.
- **중복 탐지 및 정리**: `obsidian/dup_cleaner.ts` → PR 자동 생성.
- **메트릭 수집**: `metrics/collect.ts` → cron (weekly) 실행, 결과를 Markdown으로 저장.
- **알림**: Slack/Discord webhook 스크립트 (`notify.ts`).
