## 현재 상태 진단
- **작업 지표**: 전체 100건 중 98건 완료, 0건 실패, 2건 정체 → 성공률 98 %
- **거짓 보고**: `false_report_count` 가 15건으로, 실제 오류가 없음에도 오탐이 발생, 알림 신뢰도 저하.
- **문서 규모**: Obsidian vault 에 8,226개의 문서, 개선 노트 915개 보유. 메타‑검색·분류·자동화 수준 낮음.

## 핵심 개선 목표 (3가지)
1. **거짓 보고 감소** – 오탐률 최소화, 알림 정확도 향상.
2. **정체 태스크 해결** – 정체된 태스크 자동 감지·재시도 메커니즘 도입.
3. **문서·노트 활용도 증대** – 메타데이터 자동 태깅·검색 최적화.

## 구체적 실행 계획
### 목표 1: 거짓 보고 감소
- 기존 false report 로직 분석 → 기준값 조정 및 히스토리 기반 이상치 감지 추가.
- 테스트 케이스 추가: 정상 상황에서 false_report_count 가 0이 되도록 검증.
- 배포 후 1주일간 false_report_count 모니터링.
### 목표 2: 정체 태스크 해결
- `tasks_stuck` 감지 시 자동 재시도 로직 구현 (max 3 retries).
- 재시도 실패 시 관리자 알림 및 로그 기록.
- 모니터링 대시보드에 정체 태스크 시각화 위젯 추가.
### 목표 3: 문서·노트 활용도 증대
- Obsidian vault에 메타데이터 자동 태깅 스크립트 작성 (예: `{{date}}`, `{{category}}`).
- 검색 인덱스 재구축 스케줄링 (주간).
- 개선 노트 템플릿 정의 및 신규 노트 자동 적용.

## 자동화 가능한 부분
- 정체 태스크 재시도 및 알림: CI 파이프라인에 배치 작업으로 자동 실행.
- 거짓 보고 기준 조정: 설정 파일(`config/false-report.yaml`)을 수정하면 자동 반영.
- 메타태깅 스크립트: `npm run obsidian:tag` 로 전체 노트에 일괄 적용.

## 다음 사이클 측정 지표
- `false_report_count` ≤ 5
- `tasks_stuck` ≤ 0
- 문서 검색 평균 응답 시간 ≤ 200ms
- 개선 노트 자동 태깅 적용 비율 ≥ 90%
