## 구체적 실행 계획
### 목표 1: False Report 방지 및 검증 체계 강화
- **T1**: 모든 변경에 대한 T1 검증 절차 도입 (파일·DB·HTTP 응답 직접 확인)
- **T2**: 기존 `false_report_count` 모니터링 스크립트에 알림 로직 추가 (Slack/Email)
- **T3**: 검증 자동화 테스트 케이스 20개 추가 (핵심 API, DB 마이그레이션, 이벤트 흐름)
- **T4**: `src/utils/validation.ts`에 `assertTruth` 헬퍼 구현 및 전역 사용

### 목표 2: 자동화된 작업 파이프라인 확장
- **T1**: `nco_task` 기반 연속 파이프라인 정의 파일(`pipeline.yml`) 도입
- **T2**: CI 단계에서 `nco_task` 파이프라인 실행 자동화 (GitHub Actions)
- **T3**: 실패 시 자동 롤백 및 재시도 로직 구현
- **T4**: 현재 93% 성공률을 98% 이상으로 끌어올리기 위한 성능 테스트 (load‑test, stress‑test) 추가

### 목표 3: 문서·노트 관리 효율성 및 가시성 향상
- **T1**: Obsidian Vault 메타데이터 자동태깅 스크립트(`vault/tagger.ts`) 작성
- **T2**: 개선 노트와 실제 코드 변경 매핑 자동화 (`src/cli/linkNotes.ts`)
- **T3**: 월별 문서 성장 리포트 생성 (`scripts/reportVault.sh`)
- **T4**: 불필요/중복 문서 정리 절차 정의 및 정기 실행 (월 1회)

## 자동화 가능한 부분
- **검증 자동화**: `assertTruth` → 파일·DB·API 검증 자동 실행
- **파이프라인 실행**: `nco_task` + GitHub Actions 연동
- **문서 태깅/링크**: `vault/tagger.ts`와 `cli/linkNotes.ts`를 정기 cron 실행

- **알림**: `false_report_count` 초과 시 Slack Webhook 자동 전송
