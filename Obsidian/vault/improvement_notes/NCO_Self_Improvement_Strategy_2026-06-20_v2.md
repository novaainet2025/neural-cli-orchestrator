## 현재 상태 진단
- **작업 지표**: 전체 100건 중 98건 완료, 0건 실패, 2건 정체 → 성공률 98 %
- **오류 보고**: `false_report_count` 가 15건으로, 실제 오류가 없음에도 오탐이 발생하고 있음. 알림 피로도와 신뢰도 저하 초래.
- **문서·노트 규모**: Obsidian vault에 8,226개의 문서와 915개의 개선 노트 보유. 메타데이터와 기록이 풍부하지만 검색·분류·자동화 수준이 낮아 활용도가 제한적.

## 핵심 개선 목표 (3가지)
1. **거짓 보고 감소** – 오탐률 최소화, 알림 신뢰도 향상.
2. **문서 메타 관리 향상** – 자동 메타 데이터 추출·태깅·분류, 검색 효율성 증대.
3. **자동화 워크플로 구축** – 반복 작업 최소화, 피드백 루프 가속화.

## 구체적 실행 계획 (각 목표별)
### 목표 1: 거짓 보고 감소
- a. 로그 수집 및 분석 파이프라인 검토
- b. 경고 임계값 재조정 및 오탐 방지 로직 추가
- c. 시뮬레이션 테스트 및 회귀 테스트 구축
- d. 모니터링 대시보드에 오탐 비율 시각화
### 목표 2: 문서 메타 관리 향상
- a. 파일 메타 데이터 자동 추출 스크립트 개발 (YAML 앞머리)
- b. 태그·키워드 자동 생성 규칙 정의
- c. Obsidian 플러그인 연동 스크립트 배포
- d. 기존 노트 메타 데이터 일괄 마이그레이션
### 목표 3: 자동화 워크플로 구축
- a. 개선 노트 생성 템플릿 자동화 (CLI 명령 제공)
- b. 정체된 작업 자동 알림·재시도 스케줄러 구현
- c. CI 파이프라인에 개선 노트 검증 단계 추가
- d. 피드백 루프 KPI 자동 수집·보고

## 자동화 가능한 부분
- `false_report_count` 로그 파싱 및 기준 적용 자동화 (Node 스크립트)
- 메타 데이터 추출·태깅 파이프라인 (Python/Rust)
- 개선 노트 템플릿 생성 CLI (`nco-improve create "제목"`)
- 정체 작업 감지 → Slack/Discord 알림 자동 전송
- KPI 대시보드 업데이트 자동화 (Grafana/Prometheus 연동)
