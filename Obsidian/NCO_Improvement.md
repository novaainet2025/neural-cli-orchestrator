## 구체적 실행 계획
### 목표 1: 정체 작업 감소 및 성공률 99% 이상
- 작업 큐 모니터링 강화: 1분 간격 Heartbeat 체크 도입
- 타임아웃 정책 도입: 30초 내 응답 없을 경우 자동 재시도 및 알림
- 실패/정체 작업 자동 재시도 로직 구현 및 로그 수집
- 성능 테스트 시나리오 추가 (load test 200동시 작업)

### 목표 2: 오탐지 보고 최소화
- `false_report_count` 로그 레벨 검토 및 중복 보고 방지 로직 추가
- 사용자 피드백 검증 단계 도입: 보고 시 자동 검증 스크립트 실행
- 보고 UI/CLI에 확인 절차 추가 (예: "이 보고를 확정하시겠습니까?")

### 목표 3: 개선 노트 관리 자동화
- Obsidian Vault에 `Improvement` 템플릿 파일 자동 생성 스크립트 작성
- 신규 개선 노트 자동 태깅(`#improvement`, `#next_cycle`)
- 주간 요약 자동 생성 (GitHub Actions 혹은 NCO 내 스케줄러 사용)
- 기존 5,779개의 노트를 메타데이터 기반 분류 및 중복 제거 스크립트 실행

## 자동화 가능한 부분
- **작업 모니터링 및 재시도**: NCO 내부 `TaskManager`에 플러그인 형태로 구현 → 자동 재시도 및 알림
- **오탐지 검증**: `false_report` 핸들러에 검증 스크립트 연결 → 자동 차단
- **노트 템플릿 & 태그**: `scripts/generate_improvement_note.ts` 실행 시 템플릿 기반 파일 생성 및 태그 삽입
- **주간 요약**: `cron` (PM2) 작업으로 `scripts/weekly_summary.ts` 실행 → Obsidian에 마크다운 요약 자동 커밋
