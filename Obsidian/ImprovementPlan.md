## 핵심 개선 목표 (3가지)
1. **허위 성공 보고 감소** – 검증 없이 완료된 작업 최소화
2. **태스크 실패·정체율 감소** – 에이전트 가용성 및 라우팅 강화
3. **지식베이스 효율화** – Obsidian 문서 관리 최적화 및 노이즈 감소

## 구체적 실행 계획
### 목표 1: 허위 성공 보고 감소
- **자동 품질 게이트 도입**: 모든 태스크 완료 전 `gemma-gate-check` 를 필수 검증 단계로 삽입
- **검증 실패 시 `false_report` 로 롤백**: 검증 실패 시 해당 태스크를 `false_report` 상태로 전환하고 재시도 워크플로우 트리거
- **검증 로그 중앙집중**: SQLite `task_verification` 테이블에 검증 결과 기록, 대시보드에 시각화

### 목표 2: 태스크 실패·정체율 감소
- **실패 원인 자동 분류**: 타임아웃, CLI 오류, 외부 서비스 오류 등 카테고리별 라벨링 자동화
- **스마트 라우터 강화**: 실패 시 대체 에이전트(`openrouter`, `copilot`) 로 자동 재시도
- **정체 태스크 TTL 정책**: `tasks_stuck` > 1시간 시 자동 알림 및 강제 종료 후 재큐
- **에이전트 모니터링**: `mlx` 오프라인 알림을 Slack/Webhook 으로 전달, 대체 에이전트 활성화

### 목표 3: 지식베이스 효율화
- **중복·구버전 문서 아카이브**: 30일 이상 수정되지 않은 문서는 `archive/` 로 이동
- **문서 메타데이터 정제**: `last_modified` 기반 인덱스 재구축, 최신 500개만 컨텍스트 제공
- **개선 노트 자동 집계**: `improvement_notes` 를 주간 보고서에 자동 포함, 중복 항목 병합

## 자동화 가능한 부분
- **품질 게이트 CI**: GitHub Actions 로 `npm run test && ./cli/gemma-gate-check.sh` 실행
- **실패 원인 라벨링**: `task_failed` 이벤트 발생 시 Lambda 함수가 원인 분석 후 DB 업데이트
- **정체 태스크 정리**: cron 잡(`*/5 * * * *`) 이 `tasks_stuck` > 1h 인 레코드 삭제·재큐
- **문서 아카이브 스크립트**: `node scripts/archive-obsolete-docs.ts` 로 30일 미사용 문서 이동
