## 핵심 개선 목표 (3가지)

1. **작업 안정성 및 회복력 강화** – 멈춰 있는 작업을 최소화하고 실패 복구 메커니즘을 자동화
2. **오류 보고 정확도와 신뢰성 향상** – false report를 감소시켜 신뢰성 높은 검증 체계 구축
3. **지식 관리 자동화** – Obsidian 문서와 개선 노트를 구조화·자동화하여 정보 접근성 및 재활용성 향상
1. **정체 및 실패 태스크 최소화** – 현재 정체·실패 없음 유지 및 예방
2. **오류 보고 감소** – 현재 2회 보고, 0회 목표
3. **문서·노트 관리 효율화** – 급증하는 문서·노트 자동 정리 및 검색 최적화

## 구체적 실행 계획 (각 목표별)

### 1️⃣ 작업 안정성 및 회복력 강화
- **실시간 모니터링**: `src/core/eventBus.ts`에 작업 상태 이벤트(`task:stuck`, `task:failed`)를 구독하고 Redis에 알림 기록
- **자동 재시도 로직**: `src/agent/OrchestratedLoop.ts`에 재시도 정책 적용 (exponential backoff, max 3 retries)
- **스택 트레이스 저장**: 실패 시 에러와 스택을 SQLite `task_errors` 테이블에 기록하도록 `src/utils/logger.ts` 확장
- **테스트**: `tests/agent-retry.test.ts` 추가 – 실패 시 재시도와 정상 복구 검증

### 2️⃣ 오류 보고 정확도와 신뢰성 향상
- **오류 분류 체계**: `src/utils/errorClassifier.ts` 도입 – 오류 유형 별 (`transient`, `validation`, `critical`) 라벨링
- **False Report 검증**: `src/core/FalseReportGuard.ts` 구현 – 보고 전 검증 로직 추가 (예: 중복 보고 방지, threshold 기반 차단) 
- **대시보드 UI**: `src/server/monitor.ts`에 false report 현황 시각화 추가
- **테스트**: `tests/false-report-guard.test.ts` – 정상/오류 케이스 검증

### 3️⃣ 지식 관리 자동화
- **문서 메타데이터 자동 태깅**: `src/mcp/obsidianSync.ts`에 파일 생성·수정 시 자동으로 `#task`, `#issue` 태그 삽입
- **Improvement Note 집계**: 주기적 스크립트(`scripts/aggregate-improvements.ts`)가 `improvement_notes` 디렉터리의 마크다운을 하나의 인덱스 파일로 병합
- **검색 인덱스 빌드**: `ripgrep` 기반 인덱스 (`obsidian/search-index.json`) 자동 갱신 – 빠른 키워드 검색 지원
- **CI 체크**: PR 빌드 단계에서 신규 문서가 올바른 메타데이터를 갖추었는지 검증

## 자동화 가능한 부분

- **작업 재시도 및 오류 기록**: `OrchestratedLoop`에 내장 → 별도 운영 개입 불필요
- **False Report Guard**: API 레이어에서 자동 차단 → 인간 개입 최소화
- **Obsidian 메타태그 삽입**: 파일 I/O Hook으로 자동 적용
- **Improvement Note 집계**: cron (`npm run aggregate-improvements`) 로 매일 실행
- **대시보드 업데이트**: WebSocket을 통한 실시간 푸시
- **데이터 수집**: `npm run stats` 로 JSON 지표 수집 후 마크다운 템플릿에 삽입
- **보고서 생성**: GitHub Action에서 매일/주간 자동 실행 → `NCO_Improvement_Strategy.md` 업데이트
- **알림**: 지연/실패 발생 시 webhook → Slack
