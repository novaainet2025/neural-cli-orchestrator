## 핵심 개선 목표 (3가지)
1. **성공률 향상** – 실패/정체 태스크 비율을 30% 이하로 감소.
2. **자동화 및 가시성 강화** – 오류 저장, 재시도, 모니터링 파이프라인 자동화.
3. **지식 관리 효율화** – Obsidian 노트와 개선 노트 연계, 중복 감소 및 검색 최적화.

## 구체적 실행 계획
### 목표 1: 성공률 향상
- **원인 분석**: 실패 로그 집계 및 카테고리화 (네트워크, 타임아웃, 코드 오류).
- **재시도 메커니즘**: `task-queue.ts`에 지수 백오프 재시도 로직 추가.
- **리소스 제한**: 각 에이전트당 동시 실행 제한을 `setMaxListeners` 로 명시적으로 설정.
- **테스트 확대**: 실패 시나리오 별 유닛/통합 테스트 30% 증가.

### 목표 2: 자동화 및 가시성 강화
- **오류 저장 개선**: `task-queue.ts`에서 오류 객체를 JSONL 형태로 `logs/errors/`에 저장.
- **대시보드**: Fastify 엔드포인트 `/metrics`에 현재 태스크 상태와 재시도 카운트 제공.
- **알림**: Slack/Webhook 연동으로 정체/실패 급증 시 자동 알림.
- **CI 파이프라인**: `npm run test:run` 후 성공률 기준(>70%) 미충족 시 빌드 차단。

### 목표 3: 지식 관리 효율화
- **Obsidian 연동 스크립트**: `scripts/sync-improvements.ts` → 개선 노트를 자동으로 `Obsidian_NCO_Improvement_Strategy_*.md`에 머지。
- **중복 감지**: 기존 2,548 문서 중 중복 키워드 5% 이상 식별 후 통합。
- **템플릿**: 신규 개선 노트용 표준 템플릿 정의 및 자동 적용。

## 자동화 가능한 부분
- **태스크 재시도** – 코드 레벨 자동 적용。
- **오류 로깅** – 파일 쓰기 자동화, 로그 로테이션。
- **메트릭 수집** – Prometheus exporter 구현。
- **노트 동기화** – GitHub Action 또는 `npm run sync-notes` 스크립트。
- **알림** – CI/CD 파이프라인에 Slack 알림 플러그인 연결。
