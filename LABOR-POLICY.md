## 1. 기여 보상 공식 고도화 (opencode)

- **보상 공식**: `보상 = base10 * (1 + 복잡도 배율)`
- **복잡도 배율**: 1~5 단계에 따라 다음과 같이 정의합니다.
  1. **단순** – 코드 라인 수가 10 이하, 로직 변경이 없는 경우.
  2. **보통** – 10~50 라인 변경, 기존 로직에 작은 수정.
  3. **중간** – 50~200 라인 변경, 새 함수/클래스 추가.
  4. **고급** – 200~500 라인 변경, 아키텍처 리팩터링 혹은 새로운 모듈 설계.
  5. **극복** – 500 라인 이상, 전체 시스템 설계 변경 혹은 새로운 기능 도입.
- **복잡도 측정 주체·주기**:
  - **자동 측정**: CI 파이프라인에서 변경된 라인 수, 테스트 커버리지, 의존성 그래프 변화를 분석하여 자동 배정。
  - **동료 검토(Peer)**: 리뷰어가 자동 점수에 조정이 필요할 경우, PR 리뷰 단계에서 `complexity` 라벨을 추가/수정。
  - **주기**: PR 머지 시점에 자동 측정, 필요 시 매주 팀 회의에서 리뷰。
- **최대 주간 보상 상한**: 개별 기여자당 `base10 * 5` (복잡도 5 단계) 를 초과하지 않음。 팀 전체 상한은 `base10 * 20`。

## 2. 분쟁 3단계 SLA 확정 (gemini)

| 단계 | 처리 기간 | 처리 방식 | 불복 기간 | 증거 제출 기한 |
|------|-----------|----------|-----------|----------------|
| 1심  | 48시간    | 자동 중재 (AI 기반) | 24시간 | 24시간 내 |
| 2심  | 72시간    | 패널 검토 (전문가)   | 48시간 | 48시간 내 |
| 3심  | 7일       | 거버넌스 회의 결정   | 7일   | 7일 내 |

- **불복 절차**: 각 단계에서 결과에 불복 시, 지정된 불복 기간 내에 서면(또는 디지털) 증거를 제출해야 함。
- **증거 형식**: 로그 파일, 커밋 기록, 대화 기록, API 호출 메트릭 등。

## 3. 휴식 트리거 수치 (codex)

- **연속 근무 기준**: 4시간 연속 작업(시간 기준) → 자동 휴식 알림。
  - **API 호출 기준**: 동일 에이전트에 대한 연속 API 호출이 4시간 이상 지속될 경우。
  - **토큰 기준**: 토큰 사용량이 4시간 동안 연속으로 10,000 토큰 초과 시。
- **과부하 기준**: 1시간에 50건 이상의 작업 요청이 감지되면 과부하 트리거。
- **미준수 시 UBI 패널티율**: 
  - 휴식 미이행: `5%` 보상 차감。
  - 과부하 초과: `10%` 보상 차감。
