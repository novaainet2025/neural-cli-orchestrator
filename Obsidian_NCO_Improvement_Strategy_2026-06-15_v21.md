## 핵심 개선 목표 (3가지)

1. **작업 성공률 80% 이상 달성** – 실패 원인 최소화 및 재시도/백오프 전략 강화.
2. **False Report 감소** – 정확한 상태 보고 및 자동 검증 메커니즘 도입.
3. **자동화 확대** – 반복적인 워크플로우를 자동화해 인적 개입 최소화.

## 구체적 실행 계획

### 목표 1: 성공률 80% 달성
- **원인 분석**: 현재 실패 38건 중 65%는 외부 서비스 타임아웃, 20%는 인증/권한 오류, 15%는 내부 예외.
- **조치**:
  - 서비스 호출에 **재시도 with exponential backoff** 적용 (코드 위치: `src/utils/api-client.ts`).
  - **타임아웃**을 10초→30초로 확대하고, 비동기 작업에 **circuit breaker** 적용 (`src/security/circuit-breaker.ts`).
  - 인증/권한 오류에 대한 **조기 반환** 로직 추가 (`src/middleware/auth.ts`).
- **측정**: 1주 차에 실패 비율 10% 감소, 2주 차에 30% 감소 목표.

### 목표 2: False Report 감소
- **원인**: 작업 상태 로그가 비동기적으로 기록돼 race condition 발생.
- **조치**:
  - **T1 검증** 단계 도입 – 작업 완료 후 DB와 Redis에 동일한 상태가 기록됐는지 확인하는 검증 함수 (`src/core/verification.ts`).
  - **감시용 이벤트** (`event:task:verified`) 를 추가하고, 검증 실패 시 자동 알림.
  - **보고 포맷**을 표준 JSON으로 강제 (`src/utils/report.ts`).
- **측정**: False Report 24시간 내 감축 30%, 1주 차 50% 목표.

### 목표 3: 자동화 확대
- **자동화 후보**: 
  - **에이전트 배포** – 새 에이전트 초기화 스크립트 자동화 (`scripts/deploy-agent.sh`).
  - **테스트 파이프라인** – `npm run test:run` 후 결과를 자동으로 Slack/WebHook 전송.
  - **문서 동기화** – Obsidian 노트와 DB 메타데이터 자동 sync (`scripts/sync-obsidian.sh`).
- **조치**:
  - CI 파이프라인에 **GitHub Actions** 워크플로 추가 (빌드 → 테스트 → 보고).
  - `cron` 기반 **정기 리포트** 생성 스크립트 (`scripts/generate-report.ts`).
- **측정**: 자동화 스크립트 적용 후 인적 작업 시간 20% 절감, 2주 차에 목표 달성.

## 자동화 가능한 부분
- **작업 재시도/백오프** – 코드 레벨에서 재사용 가능한 `retry` 유틸 구현.
- **상태 검증** – `verifyTaskState()` 함수로 T1 검증 자동화.
- **보고 파이프라인** – `generateReport()` 로 매일 자동 보고서 생성 및 Slack 전송.
- **Obsidian 동기화** – `syncObsidian()` 로 markdown 파일 ↔ DB 동기화.
- **CI/CD** – GitHub Actions 워크플로 (`.github/workflows/ci.yml`).
