## 개선 1: make_bar() 올림 — 저퍼센트 시각화

**문제**: 9% 사용량이 bar에서 0칸으로 표시 (사용자가 사용 중인데 빈 bar)
**원인**: 정수 나눗셈 `9*8/100=0` 절삭
**해결**: `(pct * w + 99) / 100` 올림 공식
**효과**: 1% 이상이면 최소 1칸 표시 → 시각적 피드백 즉시 제공

---

## 개선 2: 캐시 자동 갱신 — 실시간 데이터 반영

**문제**: 캐시 만료 후에도 오래된 숫자(9%)가 계속 표시됨 (실제 20%)
**원인**: statusline 스크립트가 캐시 읽기만 하고 갱신 로직 없음
**해결**: `_refresh_usage_cache()` 함수 — TTL(3분) 만료 시 OAuth API 자동 호출
**효과**: 상태바가 항상 최신 사용량 반영 (최대 3분 지연)

---

## 개선 3: 퍼센트별 색상 — 위험도 시각화

**문제**: 1일/주별 bar가 항상 고정 색상 (초록/파랑)
**해결**: `color_for_pct()` — 50% 미만 초록, 50~79% 노랑, 80%+ 빨강
**효과**: 사용량이 높아지면 색상이 변하여 rate limit 위험 사전 인지
