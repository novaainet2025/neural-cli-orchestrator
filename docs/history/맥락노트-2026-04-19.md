# NCO 맥락노트 — 2026-04-19

> 상태바(statusline) 사용량 표시 버그 수정 및 캐시 자동 갱신 추가

---

## 1. 문제 현상

### 1.1 Bar 표시 안 채워짐
- 1일 사용량 9%인데 `░░░░░░░░` (전부 빈칸)으로 표시
- 원인: `make_bar()` 정수 나눗셈 절삭 — `9 * 8 / 100 = 0`
- 1~12% 범위에서 filled=0이 되어 bar가 전혀 채워지지 않음

### 1.2 사용량 숫자 갱신 안 됨
- 실제 API 응답: 20% → 상태바 표시: 9% (1시간 전 캐시 데이터)
- 원인: `nco-statusline.sh`가 캐시 파일을 **읽기만** 하고 **갱신하지 않음**
- `anthropic-usage-bars.inc.sh`의 `_anthropic_usage_ensure_cache()` (TTL 체크 + fetch)를 source하지 않음
- 캐시 TTL 180초(3분)가 만료되어도 오래된 데이터를 계속 표시

### 1.3 색상 고정
- 1일 bar: 항상 초록, 주별 bar: 항상 파랑 (퍼센트와 무관)
- `anthropic-usage-bars.inc.sh`에는 퍼센트별 색상 로직이 있지만 `nco-statusline.sh`에서 미사용

## 2. 수정 내용

### 2.1 make_bar() 올림 공식 적용

**변경 전**: `filled=$(( pct * w / 100 ))` (절삭 → 12% 이하에서 0칸)
**변경 후**: `filled=$(( (pct * w + 99) / 100 ))` (올림 → 1%라도 최소 1칸)

| 퍼센트 | 변경 전 | 변경 후 |
|--------|---------|---------|
| 0% | ░░░░░░░░ | ░░░░░░░░ |
| 9% | ░░░░░░░░ | █░░░░░░░ |
| 13% | █░░░░░░░ | ██░░░░░░ |
| 37% | ██░░░░░░ | ███░░░░░ |
| 100% | ████████ | ████████ |

### 2.2 캐시 자동 갱신 로직 추가

`_refresh_usage_cache()` 함수 추가:
- TTL 180초 만료 시 `anthropic-usage-bars.inc.sh`를 source하여 fetch
- inc.sh가 없으면 inline으로 직접 OAuth API 호출
- OAuth 토큰: `~/.claude/.credentials.json` → macOS Keychain 폴백

### 2.3 퍼센트별 동적 색상

`color_for_pct()` 함수 추가:
- 0~49%: 초록 (GREEN)
- 50~79%: 노랑 (YELLOW)  
- 80~100%: 빨강 (RED)

### 2.4 anthropic-usage-bars.inc.sh 올림 수정

`_anthropic_usage_make_bar()`도 동일한 올림 공식으로 통일.
