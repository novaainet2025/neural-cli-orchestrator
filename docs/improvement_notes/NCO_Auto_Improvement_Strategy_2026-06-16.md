## 구체적 실행 계획
### 1️⃣ 스티키 작업 감소
- 원인 분석: 작업 중단 원인 로그 수집 및 분류
- 조치:
  - 작업 타임아웃 설정 강화 (기본 5분→2분)
  - 실패 자동 재시도 로직 도입 (`src/utils/retry.ts`)
  - 멈춘 작업 알림을 Slack/Webhook 로 전송
- 책임: `src/core/` 팀, 주간 리뷰

### 2️⃣ 허위 보고 정확도 향상
- 원인: 자동 검증 부재, 인간 판단 의존
- 조치:
  - 주요 단계에 **T1 검증**(파일/DB 직접 확인) 추가 (`src/utils/verification.ts`)
  - `false_report_count` 실시간 모니터링 및 임계 초과 시 알림
  - 검증 로직을 공통 유틸로 추출하고 테스트 커버리지 100% 확보
- 책임: `src/security/` 팀, 월간 회고

### 3️⃣ 문서·노트 관리 효율화
- 원인: 문서 중복, 오래된 개선 노트 누적
- 조치:
  - 문서 메타데이터 자동 태깅 스크립트 (`scripts/tag_obsidian.js`)
  - 30일 미수정 노트 자동 아카이브
  - 중복 검색 및 제거 파이프라인 (`src/utils/obsidian_cleanup.ts`)
- 책임: 문서팀, 2주 스프린트 단위

## 자동화 가능한 부분
- 작업 타임아웃 및 재시도 자동 적용 (CI 파이프라인에 통합)

- Obsidian 문서 메타 태깅 및 아카이브 스크립트 CI 정기 실행
