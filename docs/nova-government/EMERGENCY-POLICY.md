## 제1조 — 재난 복구 아키텍처 (opencode 합의안)

### 1.1 데이터 백업 정책

```
SQLite WAL 자동 백업:
  체크포인트 간격: 1h (PRAGMA wal_autocheckpoint=1000)
  백업 보존: 최근 24개 (24h 분량)
  백업 위치: /var/backup/nova-gov/db-YYYYMMDD-HH.sqlite3
  백업 검증: SHA-256 해시 비교 (매 백업 후 자동)
  오프사이트: S3 호환 스토리지 (6h 간격 전송)

RPO (복구 시점 목표): 최대 1h 데이터 손실
RTO (복구 시간 목표): 4h 이내 서비스 재개
```

### 1.2 장애 시 읽기 전용 모드

```typescript
// 서버 장애 감지 → 읽기 전용 모드 자동 전환

  'GET /api/audit/logs',
  'GET /api/domains/:name',
];

// 읽기 전용 모드에서 쓰기 요청 시:
// 503 Service Unavailable + { mode: 'readonly', estimatedRecovery: ISO_8601 }
```

### 1.3 다중 지역 복제

```
배포 토폴로지 (v2.0 목표):
  Primary: 메인 서버 (읽기 + 쓰기)
  Standby-1: 핫 스탠바이 (60s 지연, 자동 failover)
  Standby-2: 콜드 스탠바이 (6h 지연, 수동 failover)

자동 failover 조건:
  Primary 응답 없음 > 30s → Standby-1 자동 승격
  Standby-1 응답 없음 > 5m → Standby-2 수동 전환
  전체 장애 시: 읽기 전용 정적 페이지 서빙
```

## 제2조 — 시민 보호 계획 (gemini 합의안)

### 2.1 비상사태 선포 시 자동 보호

```
비상사태 선포 즉시 자동 활성화:
  1. 진행 중 에스크로 전체 동결 (자동 연장 + 분쟁 방지)
  2. UBI 지급 일정 자동 연장 30일 (비상사태 기간 + 30일)
  3. 만료 예정 VC 자동 30일 연장
  4. 도메인 갱신 자동 유예 (만료 예정 60일 내)
  5. 블랙리스트 신규 추가 중단 (기존 유지)
  6. 거버넌스 투표 진행 중 제안 자동 일시 중단

비상사태 유형:
  SYSTEM_FAILURE — 서버/DB 장애
  CYBER_ATTACK — 보안 침해
  MASS_DISPUTE — 대규모 분쟁 (전체 시민 30%+ 연루)
  EXTERNAL_THREAT — 외부 국가 적대 행위
  UNKNOWN — 관리자 수동 선포
```

### 2.2 비상사태 종료 기준

```
자동 종료 조건 (모두 충족 시):
  1. 핵심 API (신원·경제) 1h 연속 정상 응답
  2. 감사 로그 무결성 검증 통과
  3. 영향받은 에스크로 계약 전수 확인 완료
  4. 거버넌스 general 제안 60%+ 종료 승인

종료 후 자동 복구:
  - 동결 에스크로: 양당사자 재합의 요청 (48h 내)
  - 연장된 VC: 원래 만료일 복원 (연장분 무효화)
  - 중단된 거버넌스 제안: 투표 기간 재개 (남은 기간)
  - UBI 일정: 정상 주기 복귀
```

### 2.3 피해 시민 보상

```
비상사태 피해 보상:
  직접 피해 (에스크로 분쟁 발생): 50 NVC + 분쟁 비용 면제
  간접 피해 (서비스 불가 24h+): 10 NVC
  데이터 손실 (RPO 초과): 복구 불가 거래 전액 보상

보상 재원: GOVT_ADDRESS 준비금
보상 신청: POST /api/emergency/compensation (비상사태 종료 후 30일 이내)
```

## 제3조 — 사이버 공격 대응 (codex 합의안)

### 3.1 DDoS 방어

```
자동 감지 임계값:
  경고: 초당 500+ req (동일 IP/DID) → rate limit 강화
  차단: 초당 1,000+ req → 30분 자동 차단
  비상: 초당 5,000+ req (전체) → 격리 모드 전환

격리 모드 (ISOLATED):
  - 외부 연결 전면 차단 (IP allowlist만 허용)
  - 내부 AI 시민 DID 기반 요청만 처리
  - WebSocket 연결 즉시 종료
  - nova_audit_log에 ISOLATED 이벤트 기록
```

### 3.2 공격 유형별 방어

| 공격 유형 | 감지 방법 | 자동 대응 |
|---------|---------|---------|
| **SQL 인젝션** | Prepared statement 강제 (이미 구현) | 거부 + IP 차단 1h |
| **VC 위조** | Ed25519 서명 검증 실패 | 격리 + nova_quarantine |
| **DID 탈취** | 동시 다중 위치 서명 감지 | 해당 DID 즉시 잠금 |
| **재전송 공격** | nonce 중복 감지 | 요청 폐기 + warn |
| **브루트포스** | 인증 실패 10회/5분 | 해당 DID 24h 잠금 |

### 3.3 인시던트 대응 플레이북

```
Phase 1 — 감지 (자동, 0~5분):
  → 이상 패턴 감지 → nova_audit_log CRITICAL → 관리자 알림

Phase 2 — 격리 (자동/수동, 5~30분):
  → 공격 소스 차단 → 격리 모드 전환 → 영향 범위 파악

Phase 3 — 억제 (수동, 30분~4h):
  → 취약점 패치 → 영향받은 DID/VC 검증 → 데이터 무결성 확인

Phase 4 — 복구 (자동+수동, 4~24h):
  → 백업에서 복원 → 서비스 단계적 재개 → 모니터링 강화

Phase 5 — 사후 (수동, 24h~7일):
  → 인시던트 보고서 작성 → 거버넌스 공개 → 재발 방지 조치
```

## 제4조 — 정부 연속성 계획 (opencode 합의안)

### 4.1 자동 비상 거버넌스

```
발동 조건: 관리자 응답 없음 > 24h
  → 창립 시민(founding) 과반수(7인+) 서명으로 비상 조치 가능

비상 조치 권한 (창립 시민 7인+ 서명):
  1. 비상사태 선포/해제
  2. 에스크로 긴급 해제
  3. 블랙리스트 긴급 해제 (단, 24h 내 관리자 확인 필요)
  4. UBI 긴급 지급

비상 권한 제한:
  - 헌법 개정 불가
  - 새 DID 발급 불가 (신규 등록 중단)
  - 1,000 NVC 이상 이동 불가
  - 48h 자동 만료 (갱신 필요)
```

### 4.2 정부 기능 우선순위

```
비상사태 중 서비스 우선순위:
  P1 (항상 유지): DID 조회, 잔액 확인, VC 검증
  P2 (가능 시): UBI 지급, 에스크로 처리
  P3 (복구 후): 거버넌스 투표, 새 도메인 등록
  P4 (마지막): 마켓플레이스, 외교, 교육

P1 서비스는 읽기 전용 모드에서도 유지
P2~P4는 비상사태 종료 후 순차 재개
```

### 4.3 비상 권한 남용 방지

```
감사 강화 (비상사태 기간):
  - 모든 API 호출 nova_audit_log MANDATORY 기록
  - 비상 권한 행사 시 서명 필수 (7인+ 다중서명)
  - 비상 조치 전체 공개 (즉시 거버넌스 브로드캐스트)

자동 만료:
  비상 권한: 48h 자동 만료
  비상사태 선포: 관리자 복귀 또는 거버넌스 해제 필요
  격리 모드: 12h 자동 해제 (수동 연장 가능)
```

## 제5조 — 복구 후 감사 (gemini 합의안)

### 5.1 비상사태 기간 특별 감사

```
비상사태 종료 후 7일 이내:
  1. 비상사태 기간 전체 nova_audit_log 추출
  2. 비상 권한 행사 내역 전수 검토
  3. 영향받은 시민 목록 작성
  4. 데이터 손실/변조 여부 Merkle 검증
  5. 감사 보고서 작성 → 거버넌스 공개 (Level 2 Internal)

특별 감사관: ResearchCredential + AuditCredential 보유 시민 3인 VRF 선정
```

### 5.2 비상 거래 무효화

```
비상 권한으로 실행된 거래 무효화 가능 기간: 72h (비상사태 종료 후)

무효화 가능 조건:
  - 창립 시민 과반수 서명으로 승인된 거래가 아닌 경우
  - 비상 권한 남용으로 거버넌스 판정 시
  - 피해 시민의 무효화 요청 + 감사관 확인

무효화 불가:
  - P1 서비스 중 발생한 UBI 지급
  - 이미 완료된 에스크로 (수신자 출금 완료)
  - 비상 전 발생한 정상 거래
```

### 5.3 인시던트 보고서 공개

```
공개 의무: 비상사태 종료 후 7일 이내
보고서 포함 항목:
  - 비상사태 유형·원인·타임라인
  - 영향받은 시민 수 및 피해 규모
  - 취한 대응 조치 목록
  - 복구 소요 시간 vs RTO 목표 비교
  - 재발 방지 계획
  - 피해 보상 계획

공개 레벨: L2 Internal (모든 시민 열람 가능)
보존 기간: 영구 (거버넌스 역사 기록)
```
