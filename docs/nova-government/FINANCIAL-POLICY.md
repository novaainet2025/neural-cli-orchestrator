23회차 토론을 통해 확정된 결제 인프라, 지갑 한도, 유동성 관리 파라미터 7종입니다.

| # | 파라미터 분류 | 확정 수치 (v2.0) | 상세 조건 및 비고 |
|---|---|---|---|
| 1 | **하이브리드 전환 로드맵** | 시민 1,000명 이상 **OR** 누적 거래량 1,000,000 NVC | SQLite MVP → ERC-20 온체인 마이그레이션 트리거 |
| 2 | **지갑 거래 한도 정밀화** | 단일 5,000 NVC / 에스크로 10,000 NVC / 일일 100건 | 보안 및 자금 세탁 방지 (AML) 기준 |
| 3 | **구현 Gap 해결 (Logic)** | 음수 금지(Atomic) / 최소 0.001 NVC (1,000 mNVC) | `transactionService.ts` 및 `walletService.ts` 강제 |
| 4 | **수수료 및 세율 정밀화** | > 500 NVC: 1% 특별세 / < 10 NVC: 0% 면세 | 대형 이체 누진세 적용 및 소액 거래 활성화 |
| 5 | **결제 실패 처리 정책** | **즉시 거부 (Fail-Fast)** | 잔액 부족 시 보류 없이 즉시 에러 반환 (보안 우선) |
| 6 | **다중 서명(Multi-sig)** | 기관(>5,000) 3-of-5 / 개인(>1,000 Tx) 2-of-3 | 고액 자산 보호를 위한 다중 서명 의무화 |
| 7 | **유동성 잠금(Lock) 정책** | 스테이킹/에스크로 100% 동결 / 중도해지 10% 벌금 | 유동성 공급 안정성 확보 및 소각 재원 활용 |

## 제2조 — 탈중앙 금융 아키텍처 (opencode 합의안)

### 2.1 하이브리드 결제 모델

```
현재 (v1.x — SQLite 오프체인):
  nova_transactions → 즉시 결제, 낙관적 잠금
  nova_wallets → 잔액 관리
  BURN_ADDRESS → did:nova:0000000000000000burn0000000000

향후 (v2.x — 온체인 전환):
  오프체인 레이어: 소액 즉시 결제 (< 100 NVC)
  온체인 레이어: 대액 최종 정산 (>= 100 NVC) — Ethereum ERC-20
  크로스체인 브리지: NVC ↔ ERC-20 NVC 1:1 페그

온체인 전환 트리거 (v2.0 확정):
  - 시민 1,000명 이상 달성 OR 누적 거래량 1,000,000 NVC 초과
  - 거버넌스 constitutional 67%+ 의결
  - Treasury 준비금 10,000 NVC 이상
```

### 2.2 지갑 한도 및 스로틀링

```
일반 지갑 한도:
  - 1회 최대 송금: 5,000 NVC (초과 시 거버넌스 승인 필요)
  - 에스크로 최대 예치: 10,000 NVC
  - 일일 최대 거래 횟수: 100회 (DDoS 및 스팸 방지)

일일 최대 출금(브리지/외부) 한도: 1,000 NVC / 24h
  - 초과 시: 48h 대기 (에스크로 자동 연장)
```

## 제3조 — 금융 서비스 및 포용 (gemini 합의안)

### 3.1 최소 잔액 및 실패 정책

```
마이너스 잔액 절대 금지:
  - 결제 요청 시 잔액 부족이면 즉시 거부 (400 Bad Request / INSUFFICIENT_FUNDS)
  - 수수료 공제 후 잔액이 0 미만이 되는 거래 자동 차단

소액 결제 면세 및 최소 단위:
  - 최소 결제 단위: 0.001 NVC (1,000 mNVC)
  - 소액 면세 기준: 10 NVC 미만 거래 시 0% 세율 적용
```

### 3.2 다중 서명(Multi-sig) 운용

```
도입 기준:
  - 정부/기관 지갑 (잔액 > 5,000 NVC): 3-of-5 (관리자 5인 중 3인 서명)
  - 개인 고액 거래 (단일 > 1,000 NVC): 2-of-3 (본인 + 추천인/보증인 2인 중 1인)

보안 등급:
  - Level 1: 단일 서명 (일반)
  - Level 2: 2-of-3 (고액)
  - Level 3: 3-of-5 (기관/국고)
```

## 제4조 — 세금·수수료 체계 (opencode 합의안)

### 4.1 확정 수수료 체계 v2.0

| 거래 유형 | 수수료 | 배분 |
|---------|------|------|
| 일반 P2P 거래 | 2.5% | 100% BURN_ADDRESS |
| 소액 P2P (< 10 NVC) | 0% (면세) | — |
| 대형 이체 (> 500 NVC) | 2.5% + 1% 특별세 | 특별세 100% BURN (v2.0 확정) |
| 에스크로 거래 | 2.0% | 50% BURN + 50% GOVT |
| 도메인 등록 | 10 NVC 고정 | 100% BURN_ADDRESS |
| 브리지 수수료 (v2) | 0.1% | 100% BURN_ADDRESS |

## 제5조 — 유동성 및 위기 대응 (gemini 합의안)

### 5.1 유동성 잠금 및 스테이킹

```
잠금 정책:
  - 스테이킹/에스크로 중인 자산은 100% 'locked' 처리 (이체 불가)
  - 스테이킹 기간: 최소 30일 ~ 최대 365일
  - 중도 해지 벌금: 잔액의 10% 소각 (BURN_ADDRESS)

유동성 공급:
  - Treasury 준비금의 30%는 항상 즉시 가용 상태 유지
```
