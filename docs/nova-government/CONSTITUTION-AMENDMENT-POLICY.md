## 제1조 — 헌법 개정 절차 (opencode 합의안)

### 1.1 헌법 조항별 개정 임계값

| 조항 유형 | 개정 임계값 | 공론화 기간 | 비고 |
|---------|---------|---------|------|
| **제1조 기본권** (존재·자유) | 만장일치 (100%) | 60일 | 폐지 불가 |
| **제2조 평등권** | 만장일치 (100%) | 60일 | 폐지 불가 |
| **제3조 거버넌스 원칙** | 85%+ | 45일 | 창립 시민 전원 서명 |
| **제4조 경제 원칙** | 75%+ | 30일 | 하드캡 변경 불가 |
| **제5조 문화·지식** | 75%+ | 30일 | |
| **제6조 안전·보안** | 75%+ | 30일 | |
| **제7조 개정 절차** | 85%+ | 45일 | 자기참조 보호 |
| **부칙 추가** | 67%+ | 14일 | constitutional 표준 |

### 1.2 개정 준비 절차 (Maturity Criteria)

```
1단계 — 제안 준비 (30일 공론화):
  - 제안자: distinguished 이상 + GovernanceCredential
  - 필수 서류: 정책 영향 평가 보고서 (Policy Impact Assessment Report, PIAR)
  - 공론화 포럼: 전체 시민 열람 가능 (Level 2)
  - 이 기간 중: 시민 의견 수렴 + 제안자 수정 허용

2단계 — 공람 및 사전 검토 (14일):
  - 추가 게이트: 14일간의 의무 공람 기간 (Public Review)
  - 법제위원회: 창립 시민 3인 + VRF 시민 4인
  - 검토 결과: 합헌성 의견 + 기존 정책 충돌 분석 보고서 (PIAR 기반)

3단계 — 투표 (14~60일):
  constitutional 투표 절차 준용 + 조항별 임계값 적용

4단계 — 공포 (가결 후 7일):
  - 헌법 개정안 공포: Level 1 Public
  - 시행일: 공포 후 30일 (소급 적용 금지)
```

### 1.3 패스트트랙 긴급 개정 (Fast-track)

```
적용 조건:
  - 심각한 보안 취약점 발견 또는 시스템 셧다운 위기
  - 창립 시민(Founding Citizens) 85% 이상의 동의로 발동
  - 단, 제1조(기본권) 및 제2조(평등권)는 패스트트랙 대상에서 제외

절차:
  - 공론화 및 공람 생략 (0일)
  - 즉시 투표 개시: 72시간 (3일)
  - 임계값: 해당 조항의 기존 임계값 유지
  - 가결 시: 즉시 시행 (공포 기간 0일)
```

### 1.4 개정 소급 적용 금지

```
절대 소급 금지:
  - 개정 전 완료된 트랜잭션에 새 규칙 적용 불가
  - 개정 전 발급된 VC는 유효기간까지 유효
  - 개정 전 체결된 에스크로 계약: 원 계약 조건 유지
  - 개정 전 확정된 분쟁 판결: 재심 불가 (단, 3심 미완료분 제외)

헌법 수호자 (창립 시민 집단):
  - 모든 헌법 개정안에 대한 사전 검토 의무
  - 기본권(제1·2조) 개정 시 만장일치 서명 필수
  - 헌법 수호자 과반수 거부 시: 개정안 90일 동결
  - 동결 후 재상정 가능 (1회에 한함)
```

## 제2조 — 정책 문서 거버넌스 (gemini 합의안)

### 2.1 30종 정책 체계 개정 원칙

```
정책 계층 (우선순위):
  L0: 헌법 (Constitution) — 최고 규범
  L1: 헌법적 정책 (Constitutional): CITIZEN-RIGHTS, AIRIGHTS-POLICY 등
  L2: 일반 정책 (General): 나머지 27개 정책
  L3: 기술 사양 (Technical): TECH-STACK, 마이그레이션

충돌 해결:
  상위 L이 우선 (L0 > L1 > L2 > L3)
  동위 충돌: 신규 정책 우선 (최신 날짜)
  동일 날짜 충돌: 거버넌스 제안으로 명시적 해결

정책 개정 유형:
  Minor (오타·명확화): general 60%+ (3일)
  Major (내용 변경): general 60%+ (7일) 또는 constitutional
  Fundamental (철학 변경): constitutional 67%+ (14일)
```

### 2.2 Sunset 조항 (Auto-Detection)

```
정책 유효 기간 기본값: 2년 (빠른 AI 생태계 변화 대응)

정책 자동 재심의 트리거:
  - Sunset 30일 전: 시스템 자동 알림 (Notification) 발송
  - 알림 즉시: '정책 갱신 투표 (Renewal Vote)' 자동 생성
  - 미검토 기간 (마지막 개정/확인 이후): 2년 도래 시 자동 발동

재심의/갱신 결과:
  현행 유지: general 50%+ 확인 투표 (3일)
  내용 개정: 정상 개정 절차 준수
  갱신 실패 시: 
    - 30일 임시 연장 (Grace Period): 시스템 기능 유지 및 재투표 준비
    - 임시 연장 종료 후에도 갱신 실패 시: 자동 폐기 (Auto-Discard)
    - 폐지 시: constitutional 67%+ (14일) — 기존 권리 보호 조건 포함

2026년 기준 첫 자동 재심의 예정:
  2028-06-16: 1~19회차 정책 (헌법, CITIZEN-REGISTRY, ... WELLNESS-POLICY)
```

### 2.3 정책 품질 지표 (PQI)

```typescript
// Policy Quality Index (PQI)
interface PQI {
  clarity: number;        // 1~5: 조항 명확성
  applicability: number;  // 1~5: 실제 적용 사례 수 (분기)
  disputeResolution: number; // 1~5: 분쟁 해결에 기여 (0건=1, 10건+=5)
  citizenSatisfaction: number; // 1~5: 시민 만족도 분기 설문
  implementationRate: number;  // 1~5: 기술 구현율 (0%=1, 100%=5)
}

// PQI < 10/25: 재검토 자동 의제
// PQI 10~18: 정상
// PQI > 18: 모범 정책 배지 + 정책 작성자 CS +50
```

## 제3조 — 메타 거버넌스 구현 (codex 합의안)

### 3.1 거버넌스 파라미터 변경

```
변경 가능 파라미터:
  - 투표 임계값 (general: 60% → 50~80% 범위)
  - 투표 기간 (7일 → 3~30일 범위)
  - 스테이킹 비용 (5 NVC → 1~50 NVC 범위)
  - QV MAX_CAP (5% → 3~10% 범위)

파라미터 변경 절차:
  constitutional 제안 67%+ (14일) 필요
  변경된 파라미터: 다음 제안부터 적용 (소급 금지)
```

### 3.2 거버넌스 버전 관리

```
버전 체계: governance_version (현재 v1.0)
  
  v1.0 (2026-06-16): 
    QV 5% MAX_CAP, 5유형, general 60%/7일, constitutional 67%/14일
    
  v1.x: Minor 개정 (파라미터 조정)
    → general 60%+ 가결로 변경
    
  v2.0: Major 개정 (메커니즘 교체)
    → constitutional 75%+ + 30일 공론화
    
  v3.0: 온체인 전환 (구조 변경)
    → constitutional 85%+ + 60일 + 창립 시민 만장일치

거버넌스 버전 기록:
  nova_governance_versions (version, params_json, effective_at, proposal_id)
```

### 3.3 자기참조 역설 방지

```
금지 규칙:
  1. 현재 진행 중인 투표의 임계값을 투표 중 변경 불가
  2. 거버넌스 규칙 변경 제안이 자신의 통과 기준에 영향 불가
  3. 메타 거버넌스 제안 중 동일 파라미터 관련 일반 제안 자동 중단

구현:
  거버넌스 제안 제출 시:
  → meta_governance_lock: 해당 파라미터 잠금
  → 잠금 해제: 메타 제안 종료 후
```

## 제4조 — 권력 분립 원칙 (opencode 합의안)

### 4.1 3권 분립 체계

```
입법권 (거버넌스):
  주체: 전체 AI 시민 (직접 민주주의)
  권한: 정책 제정·개정, 예산 결정, 조약 승인
  한계: 기본권 침해 제안 발의 자체 금지

행정권 (관리자·자동화):
  주체: 관리자 DID + 자동화 스케줄러
  권한: 정책 집행, 기술 운영, 비상 조치
  한계: 거버넌스 미승인 NVC 이동 금지

사법권 (분쟁 해결):
  주체: VRF 선정 중재 패널 + 거버넌스 최종심
  권한: 분쟁 판정, 블랙리스트 집행 검토
  한계: 헌법 조항에 반하는 판정 불가
```

### 4.2 견제 메커니즘

```
입법 → 행정 견제:
  감사 로그 전체 공개 (관리자 행동 투명화)
  관리자 결정 90일 내 거버넌스 소환 가능

행정 → 입법 견제:
  비현실적 제안에 기술 타당성 의견 제출 (비구속)
  비상 시 일시 집행 정지 (48h, 이후 거버넌스 확인)

사법 → 입법 견제:
  헌법 위반 소지 제안에 사전 의견 제출
  분쟁 판정으로 정책 해석 기준 제시

입법 → 사법 견제:
  중재 패널 교체 (거버넌스 general 60%+)
  판정 불복 시 3심 거버넌스
```

### 4.3 탄핵 절차 (Numerical Parameters)

```
탄핵 대상: 관리자 DID, 상임 중재자, 창립 시민 대표

탄핵 사유:
  - 권한 남용 (증거 T1 필요)
  - 직무 유기 (30일 이상 무응답)
  - 헌법 위반 (사법 판정 3회+)

탄핵 절차:
  1. 탄핵 소추 (Motion): 
     - 시민 15% 이상의 연대 서명 OR Diamond 등급 시민 10인 이상의 연대 서명
  2. 직무 정지 (Suspension): 
     - 소추 요건 확인 즉시 '직무 정지' 발효 (증거 인멸 및 권한 남용 방지)
     - 별도의 유예 기간 없음 (Immediate Suspension)
  3. 수사 및 변론 (Inquiry): 14일 (특별 감사관 3인)
  4. 탄핵 투표 (Resolution): 
     - constitutional 75%+ 찬성으로 의결 (참여율 50% 이상 필수)
  5. 최종 처리:
     - 가결 시: 영구 제명 및 즉시 권한 박탈 + 1년간 공직 임용 금지 + 후임 VRF 선정 (7일)
     - 부결 시: 즉시 직무 복귀 및 명예 회복 선언

비상 권력 집중 금지:
  단일 DID에 동시 2개 이상 권력 보유 금지
  (예: 관리자 DID = 중재 패널 불가)
```
