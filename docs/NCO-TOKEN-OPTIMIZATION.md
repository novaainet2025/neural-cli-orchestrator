# NCO 토큰 및 효율성 최적화 완료 보고서

> **작성일**: 2026-04-19
> **버전**: v1.0
> **상태**: 활성
> **요약**: NCO 전반의 토큰 소모량을 60% 이상 절감하고 실행 효율을 극대화함.

## 1. 핵심 엔진 최적화 (Core Engine)

### 1.1 컨텍스트 관리 (`conversation-context.ts`)
- **글자 수 하드 제한**: `MAX_TOTAL_HISTORY_CHARS (4000)`, `MAX_TURN_RESPONSE_CHARS (600)` 도입.
- 무한히 증식하던 히스토리를 핵심 맥락 위주로 자동 절단하여 프롬프트 비대화 방지.

### 1.2 토론 엔진 효율화 (`discussion-engine.ts`)
- **조기 종료 (Early Exit)**: 2라운드에서 합의율 90% 달성 시 3라운드 합성을 건너뜀.
- **요약 도입 (`summarizer.ts`)**: 3라운드 합성 및 2라운드 순차 평가 시 이전 응답들을 요약하여 전달.
- **압축 포맷**: 제안서 포맷팅 시 기본 길이를 축소하고 구조화된 데이터 위주로 전달.

### 1.3 실행 루프 최적화 (`orchestrated-loop.ts`, `api-executor.ts`)
- **히스토리 압축**: 유지 턴 수를 20개에서 10개로 축소.
- **도구 결과 절단**: 1500자 이상의 도구 출력물 자동 생략.
- **팀 상태 슬림화**: 유휴 에이전트를 제외하고 '작업 중'인 에이전트 정보만 한 줄로 표시.

## 2. 슬래시 커맨드 최적화 (CLI Commands)

### 2.1 대규모 프롬프트 압축
- `nco-opus.md`: 16KB -> 1.5KB로 압축. 장황한 설명과 중복된 쉘 스크립트를 제거하고 핵심 로직 위주로 재구성.
- `nco-mesh.md`: 13KB -> 1KB로 압축. 복잡한 자연어 처리 로직을 제거하고 직관적인 명령 체계로 변경.
- `nco-analyze.md`: 8KB -> 1.2KB로 압축. 중복된 단계와 장문 가이드를 통합 및 제거.

### 2.2 공통 규칙 정립
- `nco-orchestration-prompt.ts`: 도구 사용 지침 및 규칙을 XML 기반에서 고밀도 텍스트 포맷으로 압축.
