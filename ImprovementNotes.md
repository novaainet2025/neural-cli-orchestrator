## 구체적 실행 계획
### 목표 1 – False report 개선
- 로그 파이프라인 구축: NCO 로그 → SQLite → 분석 파이프라인
- 정밀 검증 규칙 추가: T1 수준 증거 기반 판단, false positive 감소
- 대시보드에 false report 추이 시각화
- 주기적 회고: 매 사이클 false report 감소율 검증

### 목표 2 – 지식 베이스 자동화
- Obsidian 문서 인덱싱 스크립트 구현 (파일 메타·링크 추출)
- 자동 정제 파이프라인: 중복 제거, 메타 데이터 보강
- 연관 문서 자동 링크 생성 (graph DB 연동)
- 개선 노트 자동 생성 템플릿 적용

### 목표 3 – 실시간 모니터링·피드백 루프
- Prometheus + Grafana 대시보드 구축 (tasks_total, completed, stuck, false_report 등)
- 알림 규칙: stuck > 0, success_rate < 99% 시 Slack/Telegram 알림
- 주기적 성능 리포트 자동生成 (Markdown) → Obsidian vault 저장
- 회고 주기: 매 2주마다 자동 회고 노트 생성

## 자동화 가능한 부분
- **로그 수집·분석**: 로그 → DB 파이프라인 자동화 (Cron)
- **문서 인덱싱·정제**: 파일 시스템 워치러와 스크립트 연동
- **대시보드 및 알림**: Prometheus 규칙 기반 자동 알림
- **주기적 리포트·노트 작성**: 스케줄러(cron)로 Markdown 자동 생성 → Obsidian vault
