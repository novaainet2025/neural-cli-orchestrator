## 구체적 실행 계획 (각 목표별)
### 1. 거짓 보고 감소
- **데이터 수집**: 모든 보고는 T1 증거(파일 상태, DB 레코드, HTTP 응답)와 함께 기록
- **검증 파이프라인**: `nco_task`(ollama) 활용해 자동 검증 스크립트 실행 → 검증 실패 시 자동 `false_report_count` 증가
- **알림**: 검증 실패 시 Slack/Webhook 알림 및 담당자 태깅
- **주간 리뷰**: `nco_gap` 분석으로 거짓 보고 비율 감시

### 2. 실패 작업 재발 방지
- **Root Cause 분석**: `nco_analyze` 로 실패 작업 로그 자동 분류
- **테스트 강화**: 실패 시 자동 생성된 테스트 케이스(`nco_task codex`) 를 기존 테스트 스위트에 추가
- **CI 연동**: `npm run test:run` 성공률 95% 이상 유지, 실패 시 PR 자동 블록
- **문서화**: 각 실패 원인과 해결 방안을 Obsidian에 `Failure_Analysis_YYYYMMDD.md` 로 기록

### 3. 자동화 비중 확대
- **워크플로우 자동화**: 반복적인 `git status`·`git pull`·`npm install`·`npm run build` 를 `nco_task` 로 스케줄링
- **보고 자동화**: 매일 `nco_task` 로 현재 지표 수집 후 Markdown 템플릿에 채워 자동 커밋
- **에이전트 협업**: `codex`(구현) ↔ `cursor-agent`(리뷰) ↔ `ollama`(검증) 의 파이프라인을 `nco_commander` 로 한 번에 실행

## 자동화 가능한 부분
- **지표 수집 및 리포팅**: `tasks_total`, `tasks_completed`, `tasks_failed`, `false_report_count` 등은 `nco_task` 로 스크립트화
- **거짓 보고 검증**: `ollama` 를 이용한 T1 증거 자동 검사
- **실패 작업 재시도**: 실패 시 `codex` 로 패치 자동 생성 후 PR 자동 생성
- **Obsidian 노트 업데이트**: `nco_task` 로 생성된 마크다운을 지정 폴더에 커밋 → Obsidian 자동 반영
